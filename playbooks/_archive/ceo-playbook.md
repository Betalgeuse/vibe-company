## 🏠 luerre AI 연결

> **기준 문서**: [company.md](./company.md)

### Vision
**Make people do what they love**
- 고객이 비즈니스에만 집중할 수 있게
- 개발자가 코딩에만 집중할 수 있게
- **CEO도 리더십에만 집중할 수 있게**

### 이 직무의 Mission
**회사가 대표 없이도 돌아가게 만들기** - Replacement Ladder를 따라 단계별로 위임하고, 궁극적으로 CEO는 비전, 전략, 핵심 채용에만 집중합니다.

### Core Values 실천

| Core Value | 이 직무에서의 실천 |
|------------|-------------------|
| **No ego, Just impact** | 내가 다 해야 한다는 생각 버리기, 위임 |
| **Don't (just) get shit done** | 바쁜 것 ≠ 성과, 임팩트 있는 일에만 집중 |

### Problem 해결 기여

| company.md Problem | CEO 기여 |
|-------------------|---------|
| Scope Creep | 문화로 정착 - "확정 안 된 건 개발 안 함" |
| 불투명한 진행 | 투명성을 핵심 가치로 팀에 전파 |
| 커뮤니케이션 오버헤드 | AX Architect 시스템 투자, 팀 구조 최적화 |

### 팀 철학 (CEO가 만드는 문화)
우리는 이런 사람을 환영합니다:
- 평생 공부만 했는데 스타트업 씬에 도전해보고 싶은 사람
- The brave dropout - 용기 있게 선택했고, 꾸준히 공부하는 사람
- 은퇴했지만 7살의 마인드셋으로 배우고 싶은 사람

## 📘 Replacement Ladder

### Dan Martell's Framework

```
┌─────────────────────────────────────────────────────┐
│  CEO의 시간을 사는 순서                              │
├─────────────────────────────────────────────────────┤
│                                                     │
│  Phase 1: Admin 위임                                │
│  → 미팅 스케줄링, 문서 관리, 경비 정리              │

│  → 대표가 클로징에 집중 가능                        │
│                                                     │
│  Phase 4: Sales 위임                                │
│  → 영업, 계약 클로징                               │
│  → 대표가 전략/채용에 집중 가능                     │
│                                                     │
│  Phase 5: Leadership만                              │
│  → 비전, 전략, 핵심 채용                           │
│  → 회사가 대표 없이도 돌아감                        │
│                                                     │
└─────────────────────────────────────────────────────┘
```

## 📅 CEO 일정 관리

### 이상적인 주간 스케줄 (Phase 0-1)

```
┌─────────────────────────────────────────────────────┐
│  CEO 주간 스케줄                                    │
├─────────────────────────────────────────────────────┤
│                                                     │
│  월요일: Planning Day                               │
│  - 09:00-10:00 주간 계획                           │
│  - 10:00-12:00 팀 Weekly                           │
│  - 14:00-18:00 딥워크 (전략/문서)                  │
│                                                     │
│  화요일: External Day                               │
│  - 09:00-18:00 고객 미팅, 영업                     │
│                                                     │
│  수요일: No Meeting Day                            │
│  - 09:00-18:00 딥워크 (개발/제품)                  │
│                                                     │
│  목요일: External Day                               │
│  - 09:00-18:00 고객 미팅, 영업                     │
│                                                     │
│  금요일: Wrap-up Day                               │
│  - 09:00-12:00 1:1, 팀 체크인                      │
│  - 14:00-16:00 주간 리뷰                           │
│  - 16:00-18:00 다음 주 준비                        │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 시간 보호 원칙

**1. No Meeting Day 지키기**
- 수요일은 외부 미팅 금지
- 딥워크에 집중
- 긴급 상황만 예외

**2. 미팅 배칭**
- 외부 미팅은 화/목에 몰기
- 연속 미팅으로 컨텍스트 스위칭 최소화

**3. 아침 루틴**
- 09:00-10:00 이메일/Slack 금지
- 가장 중요한 일 먼저

**4. 에너지 관리**
- 고에너지 시간 (오전): 전략, 창작
- 저에너지 시간 (오후): 미팅, 리뷰

### luerre AI Core Values 실천

**No ego, Just impact**
- 팀원의 의견을 경청
- 실수를 인정
- 결과로 말하기

**Don't (just) get shit done**
- 왜 하는지 설명하기

- 팀이 목적을 이해하게 하기

### 1:1 미팅 가이드

**주기**: 직접 보고자와 격주 30분

**구조**:
```markdown
## 1:1 with [이름]

### 그들의 아젠다 (먼저)
- 이번 2주 어땠어요?
- 도움 필요한 것?
- 고민되는 것?

### 나의 아젠다
- 피드백 (구체적으로)
- 기대 사항
- 회사 방향 공유

### 개인 성장
- 커리어 목표
- 필요한 지원

### 액션 아이템
- [ ] 
```

### 피드백 원칙

**SBI 모델**:
- **S**ituation: 언제, 어떤 상황에서
- **B**ehavior: 어떤 행동을 했고
- **I**mpact: 어떤 영향이 있었다

**예시**:
```
❌ "요즘 일을 잘하고 있어요"

✅ "지난주 고객 미팅에서 (Situation)
   Scope Creep을 미리 차단한 것 (Behavior)
   덕분에 프로젝트가 일정대로 진행되고 있어요 (Impact)"
```

### 의사결정 프레임워크

**Reversible vs Irreversible**

| 유형 | 특징 | 접근법 |
|------|------|--------|
| **Type 1** (비가역적) | 되돌리기 어려움 | 신중하게, 데이터 기반 |
| **Type 2** (가역적) | 되돌릴 수 있음 | 빠르게, 실험 |

**예시**:
- Type 1: 핵심 인재 채용, 투자 유치, 피벗
- Type 2: 마케팅 채널, 도구 선택, 가격 테스트

**Type 2 결정 원칙**:
> "70%의 정보로 결정하라. 90%를 기다리면 너무 늦다."

## 🚨 위기 관리

### 위기 유형 및 대응

| 위기 | 초기 대응 | 커뮤니케이션 |
|------|----------|-------------|
| **고객 이탈** | 원인 파악, 복구 시도 | 1:1 대화 |
| **팀원 퇴사** | 인수인계, 빠른 채용 | 팀 공유 |
| **현금 부족** | 비용 절감, 매출 촉진 | 투명하게 공유 |
| **품질 이슈** | 즉시 수정, 재발 방지 | 고객 사과 |
| **보안 사고** | 즉시 대응, 전문가 투입 | 법적 검토 후 공개 |

### 위기 대응 원칙

1. **빠르게 인지**: 문제를 숨기지 않기
2. **책임지기**: 핑계 대지 않기
3. **투명하게**: 팀과 고객에게 솔직하게
4. **배움**: 재발 방지책 마련
