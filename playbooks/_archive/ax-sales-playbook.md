## 🎯 The One Number: 계약 체결 수 / 월

| 항목 | 내용 |
|------|------|
| **목표** | 3건/월 |
| **왜?** | Sales의 본질 = 돈 들어오게 하기 |
| **측정** | 서명된 계약서 수 |

**보조 지표**:
- 리드 → 미팅 전환율 > 30%
- 미팅 → 계약 전환율 > 25%
- 평균 계약 금액 > ₩10,000,000
- 영업 사이클 < 30일

## 📘 R&R (책임과 역할)

### 핵심 책임

| 책임 | 상세 | 비중 |
|------|------|------|
| **리드 자격검증** | ICP 확인, 예산/타임라인 파악 | 20% |
| **첫 미팅** | 니즈 파악, 서비스 소개 | 25% |
| **견적 제안** | Architect와 협업하여 제안서 작성 | 20% |
| **계약 클로징** | 협상, 계약 체결 | 20% |
| **고객 핸드오프** | Architect에게 프로젝트 인계 | 10% |
| **업셀/크로스셀** | 기존 고객 추가 프로젝트 | 5% |

### 권한

- 가격 협상 (승인된 범위 내: ±10%)
- 계약 조건 조율 (표준 범위 내)
- 고객 미팅 주도
- 할인 승인 요청 (CEO)

### 권한 없음 (CEO 승인 필요)

- 10% 초과 할인
- 비표준 계약 조건
- 무료 PoC 제공
- 결제 조건 변경

## 📞 리드 자격검증 (Qualification)

### BANT Framework

| 요소 | 질문 | 통과 기준 |
|------|------|----------|
| **B**udget (예산) | 이 프로젝트에 예산이 얼마나 있으신가요? | ₩5,000,000 이상 |
| **A**uthority (결정권) | 이 프로젝트의 최종 결정은 누가 하시나요? | 결정권자 또는 직접 연결 가능 |
| **N**eed (필요) | 왜 이 프로젝트가 필요하신가요? | 명확한 Pain Point |
| **T**imeline (일정) | 언제까지 완료되어야 하나요? | 3개월 내 시작 의향 |

### 자격검증 스크립트

```markdown
## 첫 통화 스크립트 (10-15분)

### 인사 (1분)
"안녕하세요, [이름]님. luerre AI의 [본인이름]입니다.
[웨비나/자료/상담신청] 해주셔서 감사합니다.
잠깐 통화 괜찮으실까요?"

### 니즈 파악 (5분)
"먼저 현재 상황을 좀 여쭤봐도 될까요?"

1. "어떤 서비스/앱을 만들고 싶으신가요?"
2. "왜 지금 이 프로젝트를 시작하려고 하시나요?"
3. "이전에 개발 외주 경험이 있으신가요? 어떠셨나요?"

### BANT 확인 (5분)
4. "이 프로젝트에 대략 얼마 정도 예산을 생각하고 계신가요?" [Budget]
5. "프로젝트 결정은 [이름]님이 직접 하시나요?" [Authority]
6. "언제쯤 시작하고 싶으신가요?" [Timeline]

### 다음 단계 (2분)
"감사합니다. 말씀 들어보니 저희가 도움드릴 수 있을 것 같아요.

다음 주에 30분 정도 시간 내셔서 
저희 서비스와 진행 방식을 자세히 설명드려도 될까요?

[일정 제안]"

### 마무리
"좋습니다. [날짜/시간]에 미팅 잡아드릴게요.
캘린더 초대 보내드리겠습니다. 감사합니다!"
```

### 자격검증 결과 기록

## 🤝 첫 미팅 (Discovery Call)

### 미팅 목표

1. 고객의 진짜 Pain Point 파악
2. 프로젝트 스코프 대략적 이해
3. luerre AI의 차별점 전달
4. 다음 단계 (견적) 합의

### 미팅 구조 (30-45분)

```
┌─────────────────────────────────────────────────────┐
│  Discovery Call 구조                                │
├─────────────────────────────────────────────────────┤
│                                                     │
│  0-5분: 인사 및 아젠다 공유                          │
│                                                     │
│  5-20분: Discovery (고객 이야기 듣기)               │
│  - 현재 상황                                        │
│  - 원하는 결과                                      │
│  - 과거 경험 (실패 이유)                            │
│  - 제약 조건 (예산, 일정)                           │
│                                                     │
│  20-35분: Solution (luerre AI 소개)                │
│  - Pain Point에 맞춘 해결책                         │
│  - 케이스 스터디                                    │
│  - 프로세스 설명                                    │
│                                                     │
│  35-45분: Next Steps                               │
│  - 질문 응답                                        │
│  - 견적 일정 합의                                   │
│  - 다음 미팅 잡기                                   │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### Discovery 질문

**현재 상황:**
- "지금 이 프로젝트 어디까지 진행되셨나요?"
- "현재 가장 큰 고민은 뭔가요?"

**원하는 결과:**
- "이 프로젝트가 성공하면 어떤 모습인가요?"
- "첫 버전에 꼭 들어가야 하는 기능은 뭔가요?"

**과거 경험:**
- "이전에 외주 개발 경험이 있으시면, 어떠셨나요?"
- "그때 가장 불편했던 점이 뭐였나요?"

**제약 조건:**
- "예산은 대략 어느 정도 생각하고 계신가요?"
- "런칭 목표 시점이 있으신가요?"

### luerre AI 차별점 설명

**Scope Creep 방지:**
```
"많은 분들이 외주 개발하시면서 
'이것도 추가해주세요'가 반복되면서 
예산이 늘어나는 경험 하셨을 거예요.

저희는 모든 요구사항을 Linear라는 도구에 이슈로 만들고,
고객님이 직접 확인 버튼을 누르셔야 개발이 시작돼요.

확정되지 않은 건 개발하지 않습니다.
그래서 '당연히 포함된 거 아니에요?'가 없어요."
```

**투명한 진행 상황:**
```
"매주 어디까지 됐는지 회의 잡으실 필요 없어요.

실시간 대시보드에서 24시간 진행 상황 보실 수 있고,
각 기능별로 To Do, 진행 중, 완료가 다 표시됩니다.

'거의 다 됐어요'가 아니라 
'87% 완료, 예상 완료일 3월 15일'로 정확하게 알려드려요."
```

## 🤝 협상 및 클로징

### 일반적인 반대 의견 (Objections)

**가격이 비싸요:**
```
"충분히 이해합니다. 
비용이 중요한 요소죠.

혹시 이전에 외주 개발 비용이 
처음 예상보다 늘어난 경험 있으셨나요?

저희는 스코프가 명확하게 정의되고,
변경 시 추가 비용이 사전에 안내되기 때문에
예상치 못한 비용 증가가 없습니다.

만약 예산이 [금액]이시라면,
스코프를 조정해서 맞출 수도 있어요.
어떤 기능이 가장 중요하세요?"
```

**다른 곳과 비교 중이에요:**
```
"당연히 비교해보셔야죠.

혹시 비교하실 때 확인하시면 좋을 것들 말씀드려도 될까요?

1. Scope 변경 시 어떻게 처리되는지
2. 진행 상황을 어떻게 확인할 수 있는지
3. 실패한 경우 환불 정책이 어떤지

저희는 이 세 가지가 다 명확해요.
비교하시고 궁금한 점 있으시면 말씀해주세요."
```

**좀 더 생각해볼게요:**
```
"네, 충분히 고민하셔야죠.

혹시 결정하시는 데 필요한 추가 정보가 있을까요?
- 케이스 스터디를 더 보여드릴까요?
- 기존 고객 레퍼런스 콜을 잡아드릴까요?
- 기술적인 질문이 있으시면 담당자 연결해드릴까요?

그리고 [날짜]까지 결정해주시면
[혜택]을 드릴 수 있어요."
```

### 클로징 기법

**Assumptive Close:**
```
"그러면 계약서는 [날짜]까지 보내드리고,
프로젝트 시작은 [날짜]로 잡으면 될까요?"
```

**Summary Close:**
```
"정리하면,
- [기능 1, 2, 3] 개발
- [일정] 완료 목표
- [금액]원 비용

이 조건으로 진행하시면 될까요?"
```

**Urgency Close (적절히 사용):**
```
"다음 주부터 새 프로젝트가 시작되면
다음 슬롯이 [날짜]로 밀릴 수 있어요.

이번 주 내 결정해주시면 
[날짜] 시작으로 잡아드릴 수 있습니다."
```

## 📈 업셀/크로스셀

### 기존 고객 관리

**체크인 주기:**
- 프로젝트 진행 중: 격주 간단 체크
- 프로젝트 완료 후: 월 1회 관계 유지
- 3개월, 6개월 후: 추가 프로젝트 제안

**업셀 기회:**
- 프로젝트 도중 "이것도 추가해주세요" → 새 견적
- 프로젝트 완료 후 → 유지보수 계약
- 런칭 후 → Phase 2 개발

**업셀 스크립트:**
```
"프로젝트 런칭 축하드려요!

초기 반응이 어떠세요?
사용자 피드백 중 추가 개발이 필요한 부분이 있으신가요?
```
