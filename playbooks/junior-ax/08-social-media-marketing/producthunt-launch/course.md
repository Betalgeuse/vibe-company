# Course: ProductHunt Launch 실행 가이드

## 📚 Step-by-Step

### D-14: Hunter 섭외 (30분)

#### Hunter 찾기 (15분)

```
1. ProductHunt에서 유사 제품 검색
   예: "AI code review" 검색

2. 각 제품의 Hunter 확인
   - Hunter 프로필 클릭
   - Followers 확인 (500+)
   - 최근 활동 확인 (3개월 내)

3. 리스트 작성 (10-15명)
   | Name | Followers | Recent Hunt | Category Match |
   |------|-----------|-------------|----------------|
   | @user1 | 1.2K | 1 week ago | ✅ AI Tools |
```

#### DM 템플릿 (15분)

```
Hi [Name]! 👋

I noticed you hunted [Product Name] - great pick!

I'm launching [My Product] on ProductHunt soon.
It's a [one-line description].

Would you be interested in hunting it?

Here's a preview: [link to draft or landing page]

Happy to discuss more!

[Your Name]
```

**팁:**
- 5-10명에게 보내기 (응답률 20-40%)
- 개인화 (그들이 Hunt한 제품 언급)
- 압박 금지 (관심 있으면 연락 달라는 톤)

### D-7: 페이지 작성 (60분)

#### Tagline (10분)

**공식:** [동사] [대상] [방법] without [단점]

```
ChatGPT 프롬프트:

당신은 ProductHunt Tagline 전문가입니다.
아래 제품의 Tagline을 5개 작성해주세요.

제품: [설명]
타겟: [타겟]
핵심 가치: [가치]

각 Tagline:
- 120자 이내
- 명확한 가치 제안
- 액션 동사 시작
- "without [단점]" 포함 (선택)

예시:
"Automate code reviews in 5min without losing quality"
```

#### Description (20분)

**구조:**
```markdown
[Problem - 2문장]
[타겟]은 [고통]을 겪습니다.

[Solution - 1문장]
[제품]은 [방법]으로 해결합니다.

[How it Works - 3단계]
1. [단계 1]
2. [단계 2]
3. [단계 3]

[Features - 3개]
✨ [기능 1]: [설명]
✨ [기능 2]: [설명]
✨ [기능 3]: [설명]

[Social Proof]
Beta tested with [숫자] users.
[결과/피드백]

[CTA]
Try it free: [링크]
```

#### Gallery Upload (10분)

```
1. 이미지 5장 순서:
   - #1: Hero (메인 화면)
   - #2: Key Feature 1

   - #4: Results/Dashboard

   - 30-60초 Demo

```

### D-Day: 런칭! (24시간 Full-Time)

#### 00:00 - 01:00 (First Hour - Critical)

```
12:01 AM PST: 런칭 확인
- ProductHunt 페이지 정상 작동 확인
- 링크 클릭 → 웹사이트 정상 작동 확인

12:05 AM: First Comment 작성
- 준비한 창업자 스토리 포스트
- AMA 시작

12:10 AM: Twitter/LinkedIn 동시 포스팅
- "We're live on ProductHunt!" 포스트
- 링크 공유
- 팀/친구 태그

12:15-01:00: 댓글 응답
- 모든 댓글 즉시 응답
- 질문에 상세 답변
- 감사 표현
```

#### 01:00 - 06:00 (Golden Hours)

```
매 30분마다:
- 새 댓글 확인 및 응답
- Upvote 추세 확인
- Twitter 추가 포스팅 (Behind-the-scenes, Milestone)

Twitter 포스트 예시:
- 1시간: "First hour: 15 upvotes! 🙏"
- 3시간: "ProductHunt 팀이 우리를 발견했어요!"
- 6시간: "Top 20 진입! 계속 가자 🚀"
```

#### 06:00 - 18:00 (Steady Growth)

```
매 1시간마다:
- 댓글 체크
- Top Comment에 추가 답변
- Reddit/HackerNews 공유 (자연스럽게)

오후 (미국 시간 아침):
- 미국 사용자 활동 증가
- Engagement 다시 증가
- 추가 Milestone 트윗
```

#### 18:00 - 24:00 (Final Push)

```
미국 서부 오후 시간 (PST 12PM-6PM):
- 마지막 Engagement 푸시
- "아직 Top 10 도전 중!" 트윗
- 커뮤니티에 마지막 공유

자정 전:
- Final Rank 확인
- 스크린샷 저장
- Thank You 포스트 준비
```

## 💡 Pro Tips

### 1. Timing is Everything
```
12:01 AM PST 정확히 런칭

이유:
- 알고리즘이 시간순 정렬
- 늦으면 불리
- 1분 차이도 중요
```

### 2. First Hour = Most Important
```
첫 1시간 Momentum이 하루를 결정

목표:
- 10-20 upvotes
- 5+ comments
- First Comment 즉시 작성
```

### 3. 댓글 > Upvotes
```
댓글 응답이 더 중요

이유:
- Engagement 알고리즘 신호
- 커뮤니티 신뢰 구축
- 추가 Upvotes 유도
```

### 4. 자연스러운 Promotion
```
❌ "Upvote 부탁드립니다!"
✅ "오늘 ProductHunt 런칭했어요. 확인해주세요!"

규칙 위반 = 패널티
```
