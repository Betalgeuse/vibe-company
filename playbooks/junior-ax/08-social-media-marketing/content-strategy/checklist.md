# Checklist: Content Strategy

## ✅ Pre-Execution (시작 전)

### 준비물 확인
- [ ] Platform Selection 완료 (주력 2개 선정)
- [ ] 제품 한 줄 설명 준비
- [ ] 타겟 사용자 정의 완료
- [ ] 브랜딩 완료 (톤앤매너)

### 시간 확보
- [ ] 45분 타임박스 설정
- [ ] 방해 요소 제거
- [ ] ChatGPT/Claude 접속 준비

### 레퍼런스 준비
- [ ] 성공한 런칭 사례 3개 북마크
- [ ] 경쟁사 소셜 미디어 계정 확인
- [ ] 인기 포스트 패턴 파악

## ✅ During Execution (실행 중)

### Step 1: 스토리 정의 (10분)
- [ ] Problem (문제) 작성
- [ ] Solution (솔루션) 작성
- [ ] Journey (여정) 작성
  - [ ] 시작점
  - [ ] 과정 하이라이트
  - [ ] 현재 상태
  - [ ] 다음 계획
- [ ] Unique Angle (독특한 관점) 작성

### Step 2: 20개 아이디어 생성 (15분)
- [ ] ChatGPT 프롬프트 작성 (5분)
  - [ ] 제품 정보 입력
  - [ ] 플랫폼 정보 입력
  - [ ] 콘텐츠 유형 7가지 명시
  - [ ] 2주 (14일) 캘린더 요청
- [ ] ChatGPT 결과 검토 (5분)
- [ ] 부족한 유형 추가 요청 (5분)

### Step 3: 콘텐츠 유형 분포 확인 (5분)
- [ ] Launch Announcement: 1개
- [ ] Behind-the-Scenes: 4-5개
- [ ] Feature Highlight: 2-3개
- [ ] User Testimonial: 1-2개
- [ ] Milestone: 2개
- [ ] Learning: 2-3개
- [ ] Problem-Solution: 1-2개
- [ ] 총 20개 확인

### Step 4: 플랫폼별 매칭 (5분)
- [ ] Twitter/X: 10-12개 할당
- [ ] LinkedIn: 3-5개 할당
- [ ] Reddit: 2-3개 할당 (선정 시)
- [ ] ProductHunt: 1개 + Updates
- [ ] Best posting time 설정

### Step 5: 우선순위 설정 (5분)
- [ ] Must-Post 6-8개 표시 ⭐⭐⭐
- [ ] Should-Post 4-6개 표시 ⭐⭐
- [ ] Nice-to-Post 나머지 표시 ⭐
- [ ] Week 1/2 균형 확인

## ✅ Post-Execution (완료 후)

### 캘린더 도구 설정
- [ ] Google Sheets 생성
  - [ ] Columns: Day, Date, Type, Topic, Platform, Status, Draft
  - [ ] 20개 Row 작성
  - [ ] Conditional Formatting (Status별 색상)
- [ ] 또는 Notion Database 생성
- [ ] 또는 Markdown File 생성

### 스토리 아크 검증
- [ ] Week 1 (Pre-Launch): 기대감 조성되는가?
- [ ] Day 0 (Launch): 명확한 Launch 포스트 있는가?
- [ ] Week 2 (Post-Launch): Momentum 유지되는가?
- [ ] 전체: 연결된 스토리인가?

### 플랫폼별 최적화 검토
- [ ] Twitter/X: 짧고 감성적인가?
- [ ] LinkedIn: 전문적이고 데이터 중심인가?
- [ ] Reddit: 가치 제공 우선인가?
- [ ] ProductHunt: 커뮤니티 참여 유도하는가?

### 80/20 Rule 검증
- [ ] 80% 가치 제공 콘텐츠:
  - Behind-the-Scenes
  - Learning
  - Problem-Solution
  - Feature Education (without CTA)
- [ ] 20% Promotion 콘텐츠:
  - Launch Announcement
  - Milestone (with CTA)
  - Feature Highlight (with CTA)

## ⚠️ Common Pitfalls (흔한 실수 체크)

### 콘텐츠 실수 방지
- [ ] 모든 포스트가 "Try now!"로 끝나지 않는가?
- [ ] Feature 나열만 하지 않는가?
- [ ] 스토리가 있는가?
- [ ] 타겟 공감 가능한가?

### 플랫폼 실수 방지
- [ ] 모든 플랫폼에 똑같은 내용 복붙 계획하지 않음
- [ ] 각 플랫폼 특성 반영했음
- [ ] Reddit에 첫 포스트부터 링크 투하 계획하지 않음

### 타이밍 실수 방지
- [ ] 모든 포스트를 같은 시간에 올리지 않음
- [ ] 플랫폼별 Best time 고려했음
- [ ] Timezone 고려했음 (PST for ProductHunt)

### 지속성 실수 방지
- [ ] 2주 후 계획도 있는가?
- [ ] 매일 콘텐츠 제작 시간 확보했는가?
- [ ] Batch creation 계획 있는가?

## 🎯 FCSR Tracking (첫 시도 성공률)

### FCSR 정의
```
Content Strategy FCSR = 
  (계획대로 발행한 포스트 수) / 
  (계획한 전체 포스트 수) × 100%

Target: 80%+
```

### FCSR 80%+ 달성 조건
- [ ] 20개 중 16개 이상 포스트 발행
- [ ] Must-Post 6-8개 100% 발행
- [ ] Should-Post 4-6개 중 50% 발행
- [ ] Engagement 발생 (댓글, 좋아요)

### FCSR 측정 방법
```
Week 2 종료 후:
- 발행한 포스트 개수: [X]
- 계획한 포스트 개수: 20
- FCSR: [X/20 × 100]%

16개 이상 → FCSR 80%+ ✅
12-15개 → FCSR 60-75% (개선 필요)
11개 이하 → FCSR 55% 미만 (전략 재검토)
```
