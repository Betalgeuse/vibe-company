# Checklist: Platform Selection

## ✅ Pre-Execution (시작 전)

### 준비물 확인
- [ ] 제품 프로토타입 URL
- [ ] 브랜딩 자료 (로고, 색상, 폰트)
- [ ] 제품 한 줄 설명 작성 완료
- [ ] 타겟 사용자 정의 완료

### 시간 확보
- [ ] 30분 타임박스 설정
- [ ] 방해 요소 제거 (알림 끄기)
- [ ] 타이머 준비

### 도구 준비
- [ ] ChatGPT/Claude 접속
- [ ] ProductHunt 계정 확인
- [ ] 경쟁사/레퍼런스 리스트 3개 준비

---

## ✅ During Execution (실행 중)

### Step 1: 제품 특성 정의 (5분)
- [ ] 제품 카테고리 선택
- [ ] 타겟 사용자 정의
- [ ] 타겟 주 활동 플랫폼 파악
- [ ] 제품 핵심 강점 정리

### Step 2: 레퍼런스 분석 (15분)
- [ ] 경쟁사/유사 제품 3개 선정
- [ ] 각 제품의 주력 플랫폼 확인
- [ ] 각 플랫폼별 성과 지표 수집 (Followers, Upvotes 등)
- [ ] 공통 패턴 추출
- [ ] 인사이트 정리 (1-2문장씩)

### Step 3: 플랫폼 점수 계산 (5분)
- [ ] 5개 플랫폼 점수표 작성
  - [ ] 타겟 존재 (1-5점)
  - [ ] Promotion 가능 (1-5점)
  - [ ] 활성도 (1-5점)
  - [ ] 경쟁사 성공 (1-5점)
  - [ ] 운영 난이도 (1-5점)
- [ ] 총점 계산
- [ ] 순위 정렬

### Step 4: 최종 선정 (5분)
- [ ] 주력 플랫폼 2개 선정
- [ ] 보조 플랫폼 1개 선정 (선택)
- [ ] 선정 근거 작성 (2-3문장)
- [ ] 플랫폼별 운영 계획 초안 (1문장씩)

---

## ✅ Post-Execution (완료 후)

### 계정 준비
- [ ] 선정 플랫폼 계정 생성/확인
  - [ ] Twitter/X (선정 시)
  - [ ] LinkedIn Company Page (선정 시)
  - [ ] ProductHunt Maker Profile (선정 시)
  - [ ] Reddit 계정 (선정 시)
  - [ ] HackerNews 계정 (선정 시)

### 프로필 최적화
- [ ] 프로필 사진: 로고 업로드 (512x512px)
- [ ] 헤더/배너: 브랜드 이미지 업로드 (플랫폼별 사이즈)
- [ ] Bio/About: 한 문장 설명 작성
- [ ] 링크: 랜딩 페이지/제품 URL 연결
- [ ] Contact: 이메일/웹사이트 추가

### Twitter/X 추가 설정 (선정 시)
- [ ] Username: @[제품명] 확보
- [ ] Bio: 150자 이내 설명 + 이모지
- [ ] Pinned Tweet: 준비 (런칭 전엔 "Coming Soon")
- [ ] 알림 설정: All mentions ON
- [ ] 모바일 앱 설치 및 로그인

### LinkedIn 추가 설정 (선정 시)
- [ ] Company Page 생성
- [ ] Tagline: 120자 설명
- [ ] About: 2,000자 이내 작성
- [ ] Industry & Size 선택
- [ ] Admins 추가 (팀원 있으면)

### ProductHunt 추가 설정 (선정 시)
- [ ] Maker Profile 완성
- [ ] Twitter 연결
- [ ] Product 초안 생성 (나중에 완성)
- [ ] Hunter 후보 리스트 작성 (5-10명)

### Reddit 추가 설정 (선정 시)
- [ ] Karma 10+ 확보 (다른 포스트에 댓글)
- [ ] 타겟 서브레딧 구독 (3-5개)
  - [ ] r/SideProject
  - [ ] r/Entrepreneur
  - [ ] r/startups
  - [ ] [니치 관련 서브레딧]
- [ ] 각 서브레딧 규칙 읽기
- [ ] Self-promotion 허용 시간 확인

### 알림 설정
- [ ] 모바일 앱 알림 ON (주력 플랫폼)
- [ ] 이메일 알림 설정
- [ ] 멘션/댓글 알림 ON
- [ ] 30분 내 응답 목표 설정

## ✅ Quality Check (품질 확인)

### 선정 검증
- [ ] 타겟 사용자가 해당 플랫폼에 80%+ 존재하는가?
- [ ] Self-promotion이 허용되는가?
- [ ] 최근 1주일 내 활발한 활동이 있는가?
- [ ] 경쟁사가 성공한 사례가 있는가?

### 운영 가능성 검증
- [ ] 2-3개 플랫폼만 선정했는가? (관리 가능)
- [ ] 각 플랫폼의 특성을 이해했는가?
- [ ] 플랫폼별 금기사항을 파악했는가?
- [ ] 매일/주간 운영 시간을 확보할 수 있는가?

### 계정 준비 검증
- [ ] 모든 프로필이 브랜딩과 일관성이 있는가?
- [ ] 링크가 올바르게 작동하는가?
- [ ] 알림이 제대로 설정되었는가?
- [ ] 모바일에서 접근 가능한가?

## ⚠️ Common Pitfalls (흔한 실수 체크)

### 선정 실수 방지
- [ ] 4개 이상 플랫폼 선정하지 않았는가?
- [ ] 트렌드만 따라가지 않았는가? (타겟 고려)
- [ ] 레퍼런스 분석 없이 직감으로 선택하지 않았는가?
- [ ] 운영 난이도를 과소평가하지 않았는가?

### 계정 준비 실수 방지
- [ ] 브랜딩과 일관성 있는가?
- [ ] Bio가 너무 길거나 복잡하지 않은가?
- [ ] 링크가 작동하는가?
- [ ] Username이 제품명과 일치하는가?

### 플랫폼별 금기사항 확인
- [ ] Twitter/X: 과도한 자동화 계획하지 않음
- [ ] LinkedIn: 캐주얼한 톤 사용 계획하지 않음
- [ ] Reddit: 첫 포스트부터 링크 투하 계획하지 않음
- [ ] ProductHunt: 준비 안 된 제품 런칭 계획하지 않음

## 📊 Success Metrics (성공 지표)

### 즉시 측정 (당일)
- [ ] 30분 내 완료했는가?
- [ ] 2-3개 플랫폼 선정했는가?
- [ ] 레퍼런스 3개 분석했는가?
- [ ] 선정 근거 문서화했는가?

### 1주일 후 측정
- [ ] 계정 프로필 완성도 100%인가?
- [ ] 첫 팔로우 20-50명 확보했는가?
- [ ] 커뮤니티 참여 시작했는가? (Reddit)
- [ ] 알림 시스템 작동하는가?

### 2주일 후 측정
- [ ] 첫 포스트 발행했는가?
- [ ] Engagement 발생했는가? (Like, Comment)
- [ ] 플랫폼 알고리즘 이해했는가?
- [ ] Content Strategy 수립 완료했는가?

## 🎯 FCSR Tracking (첫 시도 성공률)

### FCSR 정의
```
플랫폼 선정 FCSR = 
  (레퍼런스 분석 없이 올바르게 선정한 플랫폼 수) / 
  (전체 선정한 플랫폼 수) × 100%
```

### FCSR 80%+ 달성 조건
- [ ] 레퍼런스 3개 이상 분석
- [ ] 점수표 기반 객관적 선정
- [ ] 타겟 사용자 검증
- [ ] 경쟁사 성공 사례 확인

### FCSR 측정 방법
```
4주 후:
- 선정한 플랫폼에서 Engagement 발생했는가?
- 선정한 플랫폼에서 타겟 사용자 확보했는가?
- 선정한 플랫폼 운영이 지속 가능한가?

2개 중 2개 성공 → FCSR 100%
2개 중 1개 성공 → FCSR 50%
```
