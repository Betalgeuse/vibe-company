# Checklist: Launch Post Creation

## ✅ Pre-Execution (시작 전)

### 준비물
- [ ] Content Strategy 완료 (Launch Day 계획)
- [ ] 제품 스토리 정리 완료
- [ ] 베타 피드백/데이터 수집
- [ ] Branding 완료 (톤앤매너)

### 시간 확보
- [ ] 60분 타임박스 설정
- [ ] ChatGPT/Claude 접속
- [ ] 레퍼런스 3개 준비 (성공한 런칭 포스트)

## ✅ During Execution (실행 중)

### Step 1: 핵심 메시지 (10분)
- [ ] Problem 정의 (1-2문장)
- [ ] Solution 정의 (1-2문장)
- [ ] Proof 수집 (데이터 + 피드백 2-3개)
- [ ] Personal Story 작성 (왜 시작했나)

### Step 2: Twitter Thread (20분)
- [ ] ChatGPT 프롬프트 작성
- [ ] 8-10 트윗 초안 생성
- [ ] Hook 검토 (첫 트윗)
- [ ] 스토리 흐름 확인
- [ ] CTA 명확성 확인 (트윗 9)
- [ ] 해시태그 추가 (2-3개)

### Step 3: LinkedIn Post (15분)
- [ ] ChatGPT 프롬프트 작성 (전문적 톤)
- [ ] 500-800자 초안 생성
- [ ] 데이터/결과 강조
- [ ] 단락 구분 (공백 라인)
- [ ] CTA + 질문 추가
- [ ] 해시태그 추가 (3-5개)

### Step 4: ProductHunt (20분)
- [ ] Tagline 작성 (120자)
- [ ] Description 작성 (500자)
  - [ ] Problem
  - [ ] Solution
  - [ ] How it Works
  - [ ] Features (3개)
  - [ ] Why Now
  - [ ] CTA
- [ ] First Comment 작성 (창업자 스토리)
  - [ ] Personal story
  - [ ] Journey (timeline)
  - [ ] Current state
  - [ ] What's next
  - [ ] AMA 유도

### Step 5: Reddit Post (5분)
- [ ] 제목 작성 (문제 중심, 호기심)
- [ ] 본문 작성
  - [ ] Context
  - [ ] Existing Solutions + 단점
  - [ ] My Solution
  - [ ] Results/Proof
  - [ ] What I Learned (가치 제공)
  - [ ] Feedback Welcome
- [ ] 링크 자연스럽게 포함
- [ ] 서브레딧 규칙 확인

## ✅ Post-Execution (완료 후)

### Hook 강화
- [ ] ChatGPT Hook 평가 요청 (1-10점)
- [ ] 8점 미만이면 재작성
- [ ] 3가지 Hook 패턴 시도
  - [ ] 놀라운 숫자
  - [ ] 공감
  - [ ] 호기심

### 플랫폼별 최적화 검증
- [ ] Twitter: 짧고 감성적인가?
- [ ] LinkedIn: 전문적이고 데이터 중심인가?
- [ ] ProductHunt: 커뮤니티 중심인가?
- [ ] Reddit: 가치 제공 우선인가?

### CTA 검증
- [ ] 액션이 명확한가?
- [ ] 링크가 작동하는가?
- [ ] 장벽 제거 문구 포함? (free, no card, etc.)
- [ ] Urgency 추가? (선택적)

### 리뷰
- [ ] 팀/친구에게 피드백 요청
- [ ] 맞춤법/문법 확인 (Grammarly)
- [ ] 링크 클릭 테스트
- [ ] 길이 확인
  - [ ] Twitter: 트윗당 280자 이하
  - [ ] LinkedIn: 800자 이하
  - [ ] ProductHunt Tagline: 120자 이하

## ✅ Quality Check (품질 확인)

### 스토리 요소
- [ ] Personal story 포함?
- [ ] Problem → Journey → Solution 흐름?
- [ ] Emotional connection 가능?

### Social Proof
- [ ] 베타 피드백 인용?
- [ ] 데이터/숫자 포함?
- [ ] 구체적인가? (추상적 표현 X)

### CTA
- [ ] 명확한 액션?
- [ ] 링크 포함?
- [ ] 가치 제안 명확?

### 플랫폼 적합성
- [ ] 각 플랫폼 특성 반영?
- [ ] 플랫폼별 금기사항 확인?
- [ ] Best practices 준수?

## ✅ Launch Day Prep

### 예약/타이밍
- [ ] ProductHunt: 12:01 AM PST
- [ ] Twitter: 오전 9-11시 (timezone)
- [ ] LinkedIn: 오전 10-12시
- [ ] Reddit: 서브레딧별 활성 시간

### 팀 동원 (선택)
- [ ] 팀원/친구에게 사전 공지
- [ ] Launch time 공유
- [ ] Upvote/Comment 요청 (자연스럽게)

### 알림 설정
- [ ] 모바일 알림 ON (모든 플랫폼)
- [ ] 30분 내 응답 준비
- [ ] DM 확인 준비

## ✅ Backup Plans

### Plan B
- [ ] 포스트 실패 시 백업 버전 준비
- [ ] 링크 작동 안 할 때 대체 링크
- [ ] Timezone 실수 대비 체크리스트

### Engagement Prep
- [ ] 댓글 응답 템플릿 준비 (Task 6)
- [ ] 자주 묻는 질문 예상 (5개)
- [ ] DM 응답 템플릿 준비

## 📊 Success Metrics

### 즉시 측정 (1시간 내)
- [ ] Twitter: Impressions 500+
- [ ] LinkedIn: Views 100+
- [ ] ProductHunt: Upvotes 10+
- [ ] Reddit: Upvotes 5+

### 24시간 측정
- [ ] Twitter: Impressions 3,000+
- [ ] LinkedIn: Views 1,000+
- [ ] ProductHunt: Upvotes 50+
- [ ] Total Sign-ups: 50+

### FCSR Tracking
```
Launch Post FCSR = 
  (목표 달성한 플랫폼 수) / 
  (런칭한 전체 플랫폼 수) × 100%

목표:
- Engagement 발생 (댓글/좋아요)
- Sign-up 유입 확인
- Positive feedback

2/2 → 100% ✅
```
