# Course: Engagement Strategy 실행 가이드

## 📚 Step-by-Step (45분 완성)

### Step 1: ChatGPT로 템플릿 생성 (30분)

#### 프롬프트

```
당신은 소셜 미디어 커뮤니티 매니저입니다.
아래 제품의 댓글/DM 응답 템플릿을 작성해주세요.

[제품 정보]
- 제품: [이름 및 설명]
- 타겟: [타겟 사용자]
- 톤앤매너: [브랜딩에서 정의한 톤]
- 핵심 가치: [주요 이점]

[템플릿 유형] 각 5개씩:
1. Positive Feedback (긍정 피드백)
2. Questions (질문)
3. Bug Reports (버그 제보)
4. Feature Requests (기능 요청)
5. Negative Feedback (부정 피드백)
6. Collaboration (협업 제안)

[요구사항]
- 짧고 친근하게 (2-3문장)
- 이모지 적절히 사용
- 질문으로 끝내기 (Engagement 유도)
- [제품명], [기능명] 등 변수 사용

[출력 형식]
## [유형]

**상황:** [댓글 예시]
**응답:**
```
[템플릿]
```
**포인트:** [핵심 포인트]
```

#### 결과 정리

```
생성된 템플릿을:
1. Google Docs에 정리
2. 유형별 분류
3. 변수 표시 ([제품명], [기능명])
4. 빠른 접근 가능하게 북마크
```

### Step 2: 응답 시스템 구축 (10분)

#### 알림 설정

**Twitter/X:**
```
Settings → Notifications
- Mentions: ON
- Replies: ON
- Direct Messages: ON
- Mobile Push: ON
- Email: ON (선택)
```

**LinkedIn:**
```
Settings → Communications
- Comments: ON
- Mentions: ON
- Messages: ON
- Push Notifications: ON
```

**ProductHunt:**
```
Settings → Notifications
- Comments on your products: ON
- Mentions: ON
- Push: ON (모바일 앱)
```

**Reddit:**
```
Settings → Notifications
- Comment replies: ON
- Post replies: ON
- Username mentions: ON
- Mobile: ON
```

#### 템플릿 접근

**Option 1: Text Expander (Mac/Windows)**
```
단축키로 템플릿 자동 입력

예시:
/thanks → "Thank you so much! 🙏 Your support means everything."
/bug → "Thanks for reporting! Looking into it now. 🐛"
```

**Option 2: Notion Quick Access**
```
Notion 페이지:
- Title: "Engagement Templates"
- Quick Link 북마크
- Mobile 접근 가능
```

**Option 3: Google Docs (간단)**
```
Google Docs:
- "Social Media Response Templates"
- Always open in tab
- Copy-paste
```

### Step 3: 응답 프로세스 정의 (5분)

#### 30분 내 응답 플로우

```
알림 받음 (즉시)
    ↓
5분 내: 읽기 + 분류
    ↓
10분 내: 템플릿 선택 + 커스터마이즈
    ↓
15분 내: 검토 + 발송
    ↓
30분 내: 완료 ✅
```

#### 댓글 분류 시스템

```
🟢 Easy (즉시 응답 가능)
- 감사 표현
- 간단한 질문
- 긍정 피드백

🟡 Medium (10분 내)
- 복잡한 질문
- 기능 요청
- 버그 제보

🔴 Urgent (즉시)
- 부정 피드백
- 서비스 중단 제보
- 보안 이슈
```

#### 미응답 방지 시스템

```
방법 1: 스프레드시트
| Timestamp | Platform | User | Type | Status | Response |
|-----------|----------|------|------|--------|----------|
| 10:00 | Twitter | @user1 | Question | ✅ | ... |
| 10:15 | Reddit | user2 | Bug | ⏳ | ... |

방법 2: Notion Database
- 자동 알림 (미응답 30분 경과 시)
- Status tracking (Pending, Responded, Closed)
```

## 🎯 실전 시나리오

### Scenario 1: Launch Day 폭주

**상황:** 댓글 50개, DM 20개 동시 발생

**대응:**
```
1. 우선순위:
   🔴 Urgent (부정 피드백, 버그)
   🟡 Medium (질문, 기능 요청)
   🟢 Easy (감사, 긍정)

2. Batch 응답:
   - 유사한 질문 → 같은 템플릿
   - 한 번에 10개씩 처리

3. 팀 동원 (있으면):
   - 역할 분담 (플랫폼별)
```

---

### Scenario 2: 부정 피드백 대응

**댓글:**
> "Signed up but the UI is confusing. Not user-friendly."

**Bad 응답:**
```
❌ "Our UI is intuitive. Maybe you need to spend more time."
(방어적, 공감 없음)
```

**Good 응답:**
```
✅ "Thanks for the honest feedback! 🙏

Sorry the UI felt confusing. We're working on improving onboarding.

Which part specifically was unclear? 
Your input will directly help us make it better.

Happy to walk you through it - DM me!"

(공감 + 개선 의지 + 해결책 제시)
```

---

### Scenario 3: Feature Request 대응

**댓글:**
> "Love the product! Any plans for mobile app?"

**응답:**
```
✅ "Thank you! 🙏 So glad you're enjoying it.

Mobile app is on our roadmap! 📱

Quick question: 
iOS or Android priority for you?
What features would be most important on mobile?

We're prioritizing based on user needs, so your input matters!"

(감사 + 로드맵 공유 + 역질문으로 니즈 파악)
```

---

### Scenario 4: Collaboration 제안

**DM:**
> "Hi, I run a newsletter with 10K subscribers. Interested in featuring you?"

**응답:**
```
✅ "Hi [Name]! 👋

This sounds great! We'd love to be featured.

Can you share:
- Newsletter link/sample
- Typical feature format
- Timeline

Let's hop on a quick call if easier: [Calendly link]

Looking forward to it! 🚀"

(관심 + 구체적 질문 + 다음 단계)
```

## 💡 Pro Tips

### 1. Emoji 사용 가이드
```
적절히:
🙏 감사
🚀 출시/성장
💡 아이디어
🐛 버그
❤️ 사랑/지지
👋 인사

과도하지 않게:
- 1-2개/응답
- 문장 끝에
```

### 2. 응답 길이
```
플랫폼별:

Twitter: 짧게 (1-3문장)
LinkedIn: 중간 (3-5문장)
ProductHunt: 상세 (5-10문장)
Reddit: 길게 OK (가치 제공)
```

### 3. 타이밍
```
Best Response Time:

Twitter: 즉시-30분
LinkedIn: 1-2시간 OK
ProductHunt: Launch Day 즉시, 이후 2-4시간 OK
Reddit: 1-2시간 (자연스러운 토론)
```

### 4. Escalation (문제 확대 시)
```
상황:
- 공개적 부정 피드백 확산
- 버그로 인한 다수 불만
- 보안/개인정보 이슈

대응:
1. 즉시 인지 트윗/포스트
2. 해결 Timeline 공유
3. DM으로 개별 대응
4. 해결 후 Update
```
