# Outcome: Reference 기반 개발

## 🎯 목표

**그냥 만들지 않고, 항상 레퍼런스부터 찾는다**

## 📊 측정 기준

| 지표 | Before | After |
|------|--------|-------|
| 코드 재작성 횟수 | 3-5회 | 0-1회 |
| 첫 시도 성공률 (FCSR) | 20-30% | 80%+ |
| 검증된 패턴 사용 | 10% | 90%+ |

## 💡 왜 중요한가?

```
Reference 없이:
아이디어 → 바로 코딩 → 문제 발생 → 다시 작성 → 또 문제 → 포기
⏱️  2-3일 소요

Reference 기반:
아이디어 → 레퍼런스 3개 찾기 (10분) → 패턴 파악 → 적용 → 성공
⏱️  1시간 소요
```

## ✅ 성공 상태

- [ ] 작업 전 항상 레퍼런스 3개+ 찾기
- [ ] 글로벌 Top 서비스 예시 확인
- [ ] 검증된 패턴 추출
- [ ] AI에게 레퍼런스와 함께 명령
- [ ] 한 번에 원하는 결과 획득

## 🎬 기대 효과

- **시간 절약:** 재작업 시간 90% 감소
- **품질 향상:** 검증된 패턴 사용
- **학습 가속:** 좋은 코드 패턴 흡수
- **FCSR 증가:** 첫 시도 성공률 80%+
