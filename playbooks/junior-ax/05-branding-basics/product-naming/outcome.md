# Outcome: Product Naming (제품명 선정)

## 🎯 목표

**기억하기 쉽고, 도메인 확보 가능한 제품명을 30분 안에 선정**

## 📊 측정 기준

| 지표 | Target |
|------|--------|
| 제품명 후보 생성 | 10개 이상 |
| 도메인 확인 시간 | 10분 이내 |
| 최종 결정 시간 | 30분 이내 |
| 도메인 확보 가능성 | .com 또는 .io 가능 |

## 💡 왜 중요한가?

```
나쁜 네이밍:
"MyAwesomeProject123" → 기억 안남 → 검색 안됨 → 실패

좋은 네이밍:
"Notion" → 짧음 → 기억 쉬움 → 브랜딩 성공
```

**좋은 제품명의 조건:**
- ✅ 발음하기 쉬움 (3음절 이내)
- ✅ 스펠링이 직관적
- ✅ 도메인 확보 가능
- ✅ 의미가 연상됨
- ✅ 검색 결과 독점 가능

## ✅ 성공 상태

- [ ] AI가 생성한 10개+ 후보 리스트
- [ ] 도메인 확인 완료 (최소 3개 가능)
- [ ] 최종 1개 선정 완료
- [ ] 도메인 구매 또는 예약 완료
- [ ] 팀/고객에게 설명 가능한 스토리

## 🎬 기대 효과

- **브랜딩 시작점:** 모든 브랜딩의 기반
- **도메인 확보:** 나중에 바꾸면 비용 큰 증가
- **팀 정렬:** 통일된 이름으로 커뮤니케이션
- **검색 가능성:** 구글 검색 시 독점 가능

## 🎨 활용 사례

### 실제 성공 사례
- **Slack:** "검색 + 로그 + 확인" 축약어 → 협업 도구 대명사
- **Stripe:** 간단하고 결제를 연상시킴
- **Notion:** "개념"을 의미, 모든 것을 담는다는 의미
- **Linear:** 직선적인 워크플로우 의미

### 피해야 할 사례
- **MyAwesomeApp:** 너무 일반적
- **SuperTech2024:** 년도 포함은 지양
- **XYZ123:** 의미 없는 조합
- **TheNextBigThing:** 너무 길고 추상적

## 📋 결정 기준

### Must-have (필수)
1. 도메인 확보 가능 (.com, .io, .ai 중 하나)
2. 발음 가능 (전화로 설명 가능)
3. 3음절 이내
4. 포지티브한 의미

### Nice-to-have (선택)
1. 브랜드 스토리 가능
2. 로고 만들기 쉬움
3. SNS 계정 확보 가능
4. 경쟁사와 차별화

## 🚨 주의사항

### 법적 리스크
- [ ] 기존 상표권 검색 (KIPRIS)
- [ ] 유사 서비스 이름 확인
- [ ] 부정적 의미 없는지 확인 (다국어)

### 기술적 리스크
- [ ] 도메인 스쿼팅 확인
- [ ] 오타 도메인 확인 (typosquatting)
- [ ] SNS 계정 확보 가능 여부
