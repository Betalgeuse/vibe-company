# Outcome: Tone & Manner (톤앤매너 설정)

## 🎯 목표

**브랜드의 말투와 표현 방식을 30분 안에 가이드 문서화**

## 📊 측정 기준

| 지표 | Target |
|------|--------|
| 가이드 문서 | 1페이지 (A4) |
| 예시 문장 | 각 카테고리당 3개 |
| Do/Don't 리스트 | 5개씩 |
| 전체 소요 시간 | 30-45분 |

## 💡 왜 중요한가?

```
톤앤매너 없이:
"혁신적인 솔루션을 제공합니다" (마케팅)
"오류 발생함" (개발자)
→ 일관성 없음, 브랜드 정체성 약함

톤앤매너 있으면:
모든 메시지가 같은 목소리
→ 브랜드 정체성 강화
```

## ✅ 성공 상태

- [ ] 브랜드 보이스 정의 (3-5개 형용사)
- [ ] 문체 스타일 가이드 (격식/캐주얼)
- [ ] Do/Don't 예시 각 5개
- [ ] 실제 적용 예시 10개
- [ ] docs/branding/tone-manner.md 문서화

## 🎬 기대 효과

- **일관성:** 모든 터치포인트에서 같은 목소리
- **신뢰도:** 전문적이면서 진정성 있음
- **차별화:** 경쟁사와 다른 커뮤니케이션
- **효율:** 콘텐츠 제작 시 빠른 의사결정

## 📋 톤앤매너 구성 요소

### 1. Brand Voice (브랜드 보이스)
```
브랜드의 성격:
- 전문적 vs 캐주얼
- 진지함 vs 유머러스
- 격식 vs 친근함
```

### 2. Writing Style (문체)
```
문장 구조:
- 짧고 간결 vs 길고 설명적
- 능동태 vs 수동태
- 직설적 vs 완곡한
```

### 3. Vocabulary (어휘)
```
단어 선택:
- 기술 용어 사용 여부
- 전문 용어 vs 일상 용어
- 한자어 vs 순우리말
```

## 🎨 활용 사례

### Slack: 친근하고 유머러스
```
✅ "Oops! Something went wrong"
❌ "Error: System failure detected"
```

### Stripe: 전문적이지만 명확
```
✅ "Your payment was processed successfully"
❌ "Transaction completed without errors"
```

### Notion: 차분하고 포용적
```
✅ "Create your perfect workspace"
❌ "Build the ultimate productivity system"
```
