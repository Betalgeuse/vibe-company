# Checklist: FCSR 측정 및 추적

> ⚠️ **매일/매주 확인사항** - 측정하지 않으면 개선할 수 없습니다!

## ✅ 매일 (작업 종료 시)

### 1. 오늘 작업 기록
```markdown
## 2026-01-17 작업 기록

1. ✅ 로그인 페이지 생성 (첫 시도 성공)
2. ❌ API 연동 (3회 재시도)
3. ✅ CSS 수정 (첫 시도 성공)
4. ❌ 배포 스크립트 (2회 재시도)
5. ✅ 테스트 코드 작성 (첫 시도 성공)

FCSR: 3/5 = 60%
```
- [ ] 모든 작업을 기록했나?
- [ ] ✅/❌ 표시했나?
- [ ] 재시도 횟수 기록했나?
- [ ] 오늘 FCSR 계산했나?

### 2. 실패 원인 분석
```
작업 2 실패 (API 연동):
- 원인: Reference 없이 시작
- 개선: 다음엔 Supabase 공식 문서 먼저 확인

작업 4 실패 (배포):
- 원인: Constraint 불명확
- 개선: 배포 환경 변수 정확히 명시
```
- [ ] 실패한 작업마다 원인 파악했나?
- [ ] 다음에는 어떻게 할지 메모했나?

## 🚨 트러블슈팅

### ❌ "FCSR이 계속 낮아요 (50% 미만)"
**원인:** 4C Framework 미적용

**해결:**
```
1주차: Reference First 집중
  → FCSR 50% → 65%

2주차: 4단계 Prompt 적용
  → FCSR 65% → 75%

3주차: Checklist 사용
  → FCSR 75% → 80%
```

### ❌ "매일 기록하기 귀찮아요"
**원인:** 수동 기록

**해결:**
```bash
# 간단한 스크립트
echo "작업명,성공여부" >> ~/fcsr-log.csv

# 주말에 한 번에 정리
```

### ❌ "어떤 작업을 '한 건'으로 셀지 모르겠어요"
**기준:**
```
✅ 한 건으로 셈:
- AI 명령 1회 = 작업 1건
- "로그인 페이지 만들어줘" = 1건

❌ 한 건 아님:
- 파일 저장, Git commit (기계적 작업)
- 오타 수정 (단순 수정)
```
