# Outcome: 4단계 Prompt Engineering

## 🎯 목표

**AI에게 명령할 때 첫 시도 성공률 80% 달성**

## 📊 측정 기준

| 지표 | Before | After |
|------|--------|-------|
| 첫 시도 성공률 (FCSR) | 20-30% | 80%+ |
| 재명령 횟수 | 3-5회 | 0-1회 |
| 프롬프트 길이 | 1-2줄 | 4단계 구조 |
| 원하는 결과 획득 | ❌ | ✅ |

## 💡 왜 중요한가?

```
애매한 명령:
"로그인 만들어줘" → 이상한 결과 → "아니 이게 아니고..." → 다시 → 또 이상함
⏱️  30분+ 소요

명확한 명령 (4단계):
[배경] + [할 일] + [제약사항] + [결과물] → 한 번에 원하는 결과!
⏱️  5분 소요
```

## ✅ 성공 상태

- [ ] 4단계 템플릿 숙지
- [ ] 매 명령마다 4단계 구조 사용
- [ ] Reference URL 첨부
- [ ] 기술 스택 명시
- [ ] 첫 시도에 원하는 결과

## 🎬 기대 효과

- **시간 절약:** 재명령 시간 90% 감소
- **FCSR 증가:** 20% → 80%
- **정확도:** AI가 정확히 이해
- **학습 효과:** 명령 패턴 체화

## 📐 4단계 구조

```
1️⃣ Context (배경)
   "현재 Next.js 14 프로젝트, TypeScript 사용 중"

2️⃣ Action (할 일)
   "Linear 스타일 로그인 페이지 생성"

3️⃣ Constraint (제약사항)
   "Supabase Auth, shadcn/ui, Magic Link만"

4️⃣ Output (결과물)
   "app/login/page.tsx 파일, 테스트 코드 포함"
```
