# ✅ Skill 사용하기 - Checklist

## 실행 전 확인

### 1. Skill 선택
- [ ] 작업 유형에 맞는 skill 확인
  - 기능 개발/구현 → `prompt-enhancer`
  - 아키텍처 리뷰 → `codex-architecture-review`

### 2. 요청 준비
- [ ] 기본 요청 내용 준비 (간단해도 OK)
- [ ] 변경하면 안 되는 것 (비목표) 생각해두기

---

## 실행 체크리스트

### prompt-enhancer 사용 시

```
Step 1: Skill 호출
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
[ ] "prompt-enhancer skill 써서 구조화해줘: [요청내용]"
    
예시:
"prompt-enhancer skill 써서 내 요청 구조화해줘:
 대시보드에 실시간 차트 추가하고 싶어"

Step 2: 질문 응답 (필요시)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
[ ] Skill이 추가 질문하면 답변
    - 어떤 차트 유형? → "라인 차트"
    - 데이터 소스? → "API에서 가져옴"
    - UI 변경 범위? → "차트 영역만"

Step 3: 결과 검토
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
[ ] <문제 정의> 정확한지 확인
[ ] <목표> 빠진 것 없는지 확인
[ ] <비목표> 적절한지 확인 ⚠️ 가장 중요!
    - "불필요한 UI 변경 금지" 포함되어 있는가?
[ ] <성공 기준> 테스트 가능한가?
[ ] <실행 계획> 합리적인가?

Step 4: 승인 또는 수정
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
[ ] 수정 필요시: "비목표에 XX 추가해줘"
[ ] OK면: "이 계획으로 진행해줘"
```

## 빠른 참조

### Skill 호출 명령어

```
# 기본 사용
prompt-enhancer skill 써서 구조화해줘: [요청]

# 축약형
prompt enhancer로 이거 구조화: [요청]

# 직접 지정
prompt-enhancer skill 사용해서 내 요청사항을 구조화해줘
```

### 필수 비목표 항목

```markdown
<비목표>
✅ 반드시 포함:
- 목표 달성을 위해 불필요한 UI 변경 금지
- 기존 동작 코드 최소 변경
- 요청 범위 외 추가 기능 개발 금지
</비목표>
```

## 트러블슈팅

| 문제 | 해결 |
|------|------|
| Skill이 안 불러와짐 | `.factory/skills/` 폴더 확인 |
| 질문이 너무 많음 | 처음부터 상세하게 요청 |
| 범위가 너무 넓음 | 비목표 항목 추가 요청 |
| 계획이 복잡함 | 단계 나눠서 진행 요청 |
