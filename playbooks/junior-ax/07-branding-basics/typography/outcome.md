# Outcome: Typography (타이포그래피)

## 🎯 목표

**브랜드 폰트를 30분 안에 선정하고 Next.js에 적용**

## 📊 측정 기준

| 지표 | Target |
|------|--------|
| 폰트 선정 | Heading + Body 2개 |
| Next.js 적용 | 완료 |
| 웹폰트 로딩 | 최적화 완료 |
| 전체 소요 시간 | 30-45분 |

## 💡 왜 중요한가?

```
폰트 없이:
시스템 기본 폰트 → 개성 없음

폰트 시스템 있으면:
일관된 타이포그래피 → 가독성 → 브랜드 정체성
```

## ✅ 성공 상태

- [ ] Heading 폰트 선정 (제목용)
- [ ] Body 폰트 선정 (본문용)
- [ ] Google Fonts에서 다운로드
- [ ] Next.js에 적용 완료
- [ ] 폰트 웨이트 설정 (400, 600, 700)
- [ ] 타이포그래피 가이드 문서화

## 🎬 기대 효과

- **가독성:** 읽기 편한 폰트
- **브랜딩:** 일관된 느낌
- **성능:** 최적화된 로딩
- **유지보수:** 쉬운 폰트 변경

## 🎨 폰트 유형

### 1. Sans-serif (산세리프)
```
예: Inter, Roboto, Open Sans
용도: 웹/앱, 현대적, 깔끔
```

### 2. Serif (세리프)
```
예: Playfair, Merriweather
용도: 전통적, 권위적, 럭셔리
```

### 3. Monospace (모노스페이스)
```
예: Fira Code, JetBrains Mono
용도: 코드, 기술적
```

## 📋 폰트 선택 기준

### Must-have (필수)
- [ ] 가독성 좋음
- [ ] 한글 지원 (필요시)
- [ ] 웹폰트 사용 가능
- [ ] 여러 웨이트 지원 (400, 600, 700)

### Nice-to-have (선택)
- [ ] Variable Font 지원
- [ ] 브랜드 컨셉과 일치
- [ ] 경쟁사와 차별화
- [ ] 무료 라이선스

## 🚨 피해야 할 것

❌ **너무 독특한 폰트:** 가독성 저하  
❌ **무거운 폰트:** 로딩 느림  
❌ **유료 폰트:** 라이선스 복잡  
❌ **한글 미지원:** 깨짐
