# Course: Tone & Manner 실행 가이드

## 📚 Step-by-Step (30-45분 완성)

### Step 1: 브랜드 보이스 정의 (10분)

#### ChatGPT 프롬프트
```
브랜딩 전문가로서 도와주세요.

[제품 정보]
- 제품: [제품명]
- 타겟: [타겟 사용자]
- 컨셉 키워드: [키워드 3개]

질문:
1. 이 브랜드의 목소리를 형용사 5개로 표현한다면?
2. 격식/캐주얼 스펙트럼에서 어디에 위치하나? (1-10)
3. 유사한 톤의 브랜드 3개는?

각 형용사마다 설명을 1문장으로 추가해주세요.
```

#### 브랜드 보이스 스펙트럼
```
다음 축에서 우리 브랜드 위치 표시:

격식적 ----●---- 캐주얼
진지함 --------● 유머러스  
전문적 ●-------- 친근함
권위적 ------●-- 포용적
```

---

### Step 2: 문체 가이드 작성 (10분)

#### 문장 구조
```markdown
## 문장 길이
✅ Do: 짧고 명확하게 (15단어 이내)
❌ Don't: 길고 복잡한 문장

## 어조
✅ Do: 능동태 사용 ("우리가 해결합니다")
❌ Don't: 수동태 ("문제가 해결되었습니다")

## 인칭
✅ Do: "우리"와 "당신" 사용
❌ Don't: "본 서비스", "고객님"
```

#### 어휘 선택
```markdown
## 기술 용어
✅ Do: 필요시에만, 설명 추가
❌ Don't: 과도한 전문 용어

## 외래어
✅ Do: 보편적인 것만 (API, MVP)
❌ Don't: 불필요한 외래어 남발

## 완곡 표현
✅ Do: 직설적이되 정중하게
❌ Don't: 지나치게 돌려 말하기
```

---

### Step 3: Do/Don't 예시 작성 (10분)

#### 카테고리별 예시

**1. 에러 메시지**
```
✅ Do: "파일을 업로드할 수 없습니다. 파일 크기는 10MB 이하여야 합니다."
❌ Don't: "업로드 실패: ERR_FILE_SIZE_EXCEEDED"
```

**2. 성공 메시지**
```
✅ Do: "완료했습니다! 이메일을 확인해주세요."
❌ Don't: "요청이 성공적으로 처리되었습니다."
```

**3. 안내 문구**
```
✅ Do: "시작하려면 이메일을 입력하세요."
❌ Don't: "본 서비스 이용을 위해 이메일 주소를 기입해주시기 바랍니다."
```

**4. CTA (Call-to-Action)**
```
✅ Do: "무료로 시작하기"
❌ Don't: "서비스 가입 신청"
```

**5. 설명 문구**
```
✅ Do: "AI가 코드를 분석해서 개선점을 알려드립니다."
❌ Don't: "인공지능 기반의 코드 품질 분석 솔루션을 제공합니다."
```

### Step 4: 실제 적용 예시 (10분)

#### 웹사이트 주요 문구

**히어로 섹션:**
```
✅ 좋은 예:
"더 나은 코드를 작성하세요"
"AI 코드 리뷰로 팀 생산성 2배 증가"

❌ 나쁜 예:
"혁신적인 AI 기술로 코드 품질을 획기적으로 개선"
```

**기능 설명:**
```
✅ 좋은 예:
"버그 발견
AI가 숨은 버그를 찾아냅니다."

❌ 나쁜 예:
"Bug Detection
고급 알고리즘을 통한 잠재적 오류 탐지 기능"
```

**가격 페이지:**
```
✅ 좋은 예:
"무료로 시작
신용카드 불필요"

❌ 나쁜 예:
"Free Plan Available
No Credit Card Required"
```

### Step 5: 가이드 문서 작성 (5분)

#### 템플릿
```markdown
# Tone & Manner Guide: [브랜드명]

## Brand Voice
우리는 [형용사 5개]한 목소리로 말합니다.

1. **[형용사 1]**: [설명 1문장]
2. **[형용사 2]**: [설명 1문장]
3. **[형용사 3]**: [설명 1문장]

## Writing Principles
1. [원칙 1]: [설명]
2. [원칙 2]: [설명]
3. [원칙 3]: [설명]

## Do's and Don'ts

### ✅ Do
- [Do 1]
- [Do 2]
- [Do 3]

### ❌ Don't
- [Don't 1]
- [Don't 2]
- [Don't 3]

## Examples

### Error Messages
✅ "[좋은 예시]"
❌ "[나쁜 예시]"

### Call-to-Actions
✅ "[좋은 예시]"
❌ "[나쁜 예시]"

## Voice Spectrum
격식적 ----●---- 캐주얼
진지함 --------● 유머러스  

## Vocabulary
사용: [단어 리스트]
피하기: [단어 리스트]
```

## 💡 Pro Tips

### 1. 경쟁사 분석
```
경쟁사 3개의 웹사이트 메인 문구 수집:
- 문장 길이
- 어조 (격식/캐주얼)
- 사용 단어

→ 차별화 포인트 찾기
```

### 2. A/B 테스트로 검증
```
같은 메시지를 2가지 톤으로:
A: "회원가입하세요"
B: "무료로 시작하기"

→ 어느 것이 더 브랜드에 맞는가?
```

### 3. 실제 콘텐츠에 즉시 적용
```
가이드 작성 직후:
1. 랜딩 페이지 주요 문구 리라이팅
2. 에러 메시지 전체 수정
3. 이메일 템플릿 업데이트
```
