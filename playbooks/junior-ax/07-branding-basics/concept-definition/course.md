# Course: Concept Definition 실행 가이드

## 📚 Step-by-Step (30-45분 완성)

### Step 1: 레퍼런스 수집 (10분)

#### 찾아야 할 레퍼런스
```
1. 직접 경쟁사 (같은 영역)
2. 간접 경쟁사 (다른 영역, 비슷한 타겟)
3. 벤치마크 (완전히 다른 영역, 부러운 브랜딩)
```

#### 레퍼런스 찾는 곳
```
1. Mobbin (https://mobbin.com) - 앱 디자인
2. Dribbble (https://dribbble.com) - 브랜딩
3. Behance (https://behance.net) - 브랜드 아이덴티티
4. 직접 경쟁사 웹사이트
```

#### 실행 방법
```bash
# 예: AI 코드 리뷰 도구를 만든다면

직접 경쟁사:
1. CodeRabbit (코드 리뷰 AI)
2. GitHub Copilot (AI 코딩 도구)

간접 경쟁사:
3. Linear (이슈 트래커, 개발자 타겟)

벤치마크:
4. Stripe (B2B 개발자 도구)
```

#### 정보 수집
```markdown
각 레퍼런스마다 기록:

## [서비스명]
- **URL:** [링크]
- **주요 색상:** [색상 코드]
- **폰트 느낌:** [산세리프/세리프/모노스페이스]
- **키워드 3개:** [전문적, 미니멀, 기술적]
- **차별화 포인트:** [뭐가 특별한가?]
```

---

### Step 2: 패턴 분석 (10분)

#### ChatGPT 활용
```
프롬프트:

나는 [제품명]을 만들고 있습니다.
다음 3개 서비스의 브랜딩을 분석해주세요.

[서비스 1: CodeRabbit]
- 주요 색상: 보라색, 검정
- 폰트: 모던 산세리프
- 키워드: AI, 정확함, 빠름

[서비스 2: GitHub Copilot]
- 주요 색상: 파란색, 흰색
- 폰트: 시스템 폰트
- 키워드: 협업, 자동화, 개발자

[서비스 3: Linear]
- 주요 색상: 보라색, 다크 모드
- 폰트: Inter (산세리프)
- 키워드: 빠름, 정확함, 아름다움

질문:
1. 공통 패턴은?
2. 각자의 차별화 포인트는?
3. 내가 피해야 할 것은?
4. 내가 취할 수 있는 독특한 방향은?
```

#### 분석 결과 예시
```
공통 패턴:
- 개발자 도구 → 다크 모드 선호
- 신뢰감 → 보라/파랑 계열
- 기술적 → 산세리프 폰트

차별화 포인트:
- CodeRabbit: AI 강조 (보라색)
- GitHub: 협업 강조 (파란색)
- Linear: 속도 강조 (애니메이션)

우리의 방향:
→ "코드 품질"에 집중
→ 차분하지만 신뢰감 있는 느낌
→ 초록-파랑 계열로 차별화
```

---

### Step 3: 키워드 도출 (10분)

#### 키워드 카테고리
```
1. 기능적 키워드 (무엇을 하는가)
   예: 정확함, 빠름, 자동화

2. 감성적 키워드 (어떤 느낌인가)
   예: 신뢰, 안정, 편안함

3. 시각적 키워드 (어떻게 보이는가)
   예: 미니멀, 모던, 전문적
```

#### 브레인스토밍
```
포스트잇 방식 (또는 메모장):

제품의 특징 단어 나열:
- 정확함, 빠름, AI, 코드, 리뷰
- 품질, 개발자, 자동화, 효율

느낌 단어 나열:
- 신뢰, 전문적, 깔끔, 모던
- 차분함, 안정적, 똑똑함

시각 단어 나열:
- 미니멀, 기술적, 세련됨
- 다크, 컬러풀, 그라데이션
```

#### 키워드 좁히기
```
9개 → 5개 → 3개

기준:
1. 제품 핵심 가치를 가장 잘 표현하는가?
2. 경쟁사와 차별화되는가?
3. 시각적으로 표현 가능한가?

최종 3-5개 선택!
```

---

### Step 4: 컨셉 문서 작성 (10분)

#### 템플릿
```markdown
# Brand Concept: [제품명]

## 브랜드 키워드
1. **[키워드 1]** - [설명 1문장]
2. **[키워드 2]** - [설명 1문장]
3. **[키워드 3]** - [설명 1문장]

## 브랜드 퍼스널리티
만약 [제품명]이 사람이라면:
- **나이:** [20대 중반, 전문가]
- **성격:** [차분하고 논리적이지만 친근함]
- **말투:** [격식 있지만 딱딱하지 않음]

## 감성 방향
사용자가 느껴야 할 감정:
- [신뢰감]: "이 도구는 내 코드를 안전하게 검토한다"
- [안정감]: "에러 없이 정확하게 작동한다"
- [효율]: "시간을 절약해준다"

## 비주얼 방향
### 색상
- **주조색:** [블루-그린 계열] - 신뢰와 성장
- **강조색:** [연한 초록] - 코드 품질 향상
- **배경:** [다크 모드 우선] - 개발자 친화적

### 폰트
- **제목:** [모던 산세리프] - 전문적
- **본문:** [Inter/Roboto] - 가독성

### 스타일
- **전체 톤:** 미니멀하지만 따뜻함
- **이미지:** 코드 중심, 사람은 부차적
- **애니메이션:** 부드럽고 자연스러움

## 차별화 포인트
경쟁사 대비:
1. [차별점 1]
2. [차별점 2]
3. [차별점 3]

## 금지 사항
이것만은 하지 말기:
- ❌ 너무 화려한 색상 (집중 방해)
- ❌ 장난스러운 톤 (신뢰 저하)
- ❌ 복잡한 디자인 (효율성 저하)

## 레퍼런스
- [서비스 1]: [URL] - [참고 이유]
- [서비스 2]: [URL] - [참고 이유]
- [서비스 3]: [URL] - [참고 이유]
```

```
질문:

```

## 🎯 실전 예시

### Case Study: Linear의 브랜드 컨셉

#### 키워드
1. **Fast** - 모든 인터랙션이 즉각 반응
2. **Precise** - 정확한 이슈 관리
3. **Beautiful** - 아름다운 UI/UX

#### 퍼스널리티
- 나이: 20대 후반, 실리콘밸리 엔지니어
- 성격: 효율 지향, 완벽주의, 미학 추구
- 말투: 간결하고 명확

#### 비주얼 방향
- 색상: 다크 모드 + 퍼플 액센트
- 폰트: Inter (깔끔한 산세리프)
- 애니메이션: 부드럽고 빠름

#### 결과
→ 개발자들이 "가장 아름다운 이슈 트래커"라고 평가
→ 사용 경험이 브랜드 컨셉과 완벽히 일치

## 💡 Pro Tips

### 1. "만약 ~라면?" 질문
```
브랜드 퍼스널리티를 명확히 하는 질문들:

- 만약 브랜드가 차라면? 
  → BMW? Toyota? Tesla?

- 만약 브랜드가 음악이라면?
  → 클래식? 재즈? 일렉트로닉?

- 만약 브랜드가 장소라면?
  → 사무실? 카페? 연구소?
```

### 2. 경쟁사 2x2 매트릭스
```
           신뢰감 높음
               |
캐주얼 --------+-------- 전문적
               |
           신뢰감 낮음

경쟁사 위치 표시 → 빈 공간 찾기
```

### 3. 무드보드 작성
```
Pinterest 또는 Figma:
- 레퍼런스 이미지 10개 수집
- 색상, 폰트, 스타일 일관성 확인
- 팀과 공유
```

### 4. 5-Why 기법
```
왜 "신뢰감"이 중요한가?
→ 사용자가 AI를 믿어야 하니까

왜 AI를 믿어야 하나?
→ 코드 리뷰가 정확해야 하니까

왜 정확해야 하나?
→ 버그가 프로덕션에 가면 안 되니까

→ 따라서 "정확함, 안정성, 신뢰"가 핵심!
```

## 🚨 흔한 실수

### ❌ 실수 1: 키워드 너무 많음
```
10개 키워드 → 방향성 흐려짐

해결:
3-5개로 집중! "적을수록 강하다"
```

### ❌ 실수 2: 추상적인 표현
```
"혁신적이고 창의적인" → 모호함

해결:
"기술적이고 신뢰할 수 있는" → 구체적
```

### ❌ 실수 3: 경쟁사 그대로 따라하기
```
"Linear가 보라색이니까 우리도" → 차별화 실패

해결:
패턴 분석 → 다른 길 찾기
```

### ❌ 실수 4: 실행 불가능한 컨셉
```
"럭셔리하고 고급스러운" → 하지만 무료 서비스?

해결:
제품 현실과 맞는 컨셉!
```
