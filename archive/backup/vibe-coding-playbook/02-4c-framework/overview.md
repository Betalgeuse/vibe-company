# 📘 The 4C Framework

**Vibe Coding의 핵심 방법론**

Vibe Coding의 핵심을 4가지 요소로 구조화합니다.


## 🍳 요리 레시피 비유

요리 레시피를 만든다고 생각하세요:

- **Camcorder (영상)**: 요리하는 모습을 영상으로 촬영
- **Course (순서)**: 1단계, 2단계... 순서대로 적은 레시피  
- **Cadence (주기)**: 언제, 얼마나 자주 만드는지
- **Checklist (체크리스트)**: 맛있게 만들어졌는지 확인 항목


## 📊 4C 구조도

```
┌─────────────────────────────────────────────────┐
│            The 4C Framework                     │
├─────────────────────────────────────────────────┤
│                                                 │
│  1. 🎥 Camcorder  → 보여주기 (영상)             │
│     "이렇게 하는 거구나!"                         │
│                                                 │
│  2. 📚 Course     → 따라하기 (매뉴얼)            │
│     "1단계, 2단계, 3단계..."                      │
│                                                 │
│  3. ⏰ Cadence    → 언제 하기 (스케줄)           │
│     "매일? 매주? 필요할 때?"                      │
│                                                 │
│  4. ✅ Checklist  → 확인하기 (품질 체크)         │
│     "제대로 했나요?"                             │
│                                                 │
└─────────────────────────────────────────────────┘
```


## 💡 왜 4C인가?

### 문제: 지식이 흩어져 있음

많은 팀에서 겪는 문제:
- "이거 어떻게 하는 거였지?" 🤔
- "누가 알려줬는데 까먹었어..." 😅
- "메모는 어디 갔지?" 📝
- "다시 물어보기 민망한데..." 😰

### 해결: 4C로 체계화

```
Before (혼란)              After (체계화)
    ↓                          ↓
말로만 전달               영상 + 문서
    ↓                          ↓
기억에 의존               언제든 확인 가능
    ↓                          ↓
매번 물어봄               스스로 해결
```


## 🎯 각 요소의 역할

### 1. Camcorder (영상) - "보여주기"
**목적**: 실제 작업 과정을 눈으로 확인
**효과**: 이해도 ↑, 실수 ↓

[자세히 보기 →](./camcorder.md)

### 2. Course (매뉴얼) - "따라하기"
**목적**: 단계별 상세 가이드 제공
**효과**: 누구나 재현 가능

[자세히 보기 →](./course.md)

### 3. Cadence (주기) - "언제 하기"
**목적**: 작업 타이밍과 빈도 정의
**효과**: 놓치는 일 없음

[자세히 보기 →](./cadence.md)

### 4. Checklist (체크리스트) - "확인하기"
**목적**: 품질 검증 기준 제시
**효과**: 완성도 보장

[자세히 보기 →](./checklist.md)


## 🔄 4C 실행 흐름

```
┌──────────────────────────────────────────┐

│      ↓                                   │

│      ↓                                   │

│      ↓                                   │
│  🔁  반복 실행 & 개선                     │
└──────────────────────────────────────────┘
```


## 📖 사용 예시

### 예시 1: 로그인 기능 구현

| 요소 | 내용 |
|------|------|
| **Camcorder** | 로그인 구현하는 전 과정 화면 녹화 |
| **Course** | 1. API 연동 → 2. UI 작성 → 3. 에러 처리 |
| **Cadence** | 신규 프로젝트마다 재사용 |
| **Checklist** | ✓ 로그인 성공? ✓ 에러 처리? ✓ 보안 체크? |

### 예시 2: 코드 리뷰 프로세스

| 요소 | 내용 |
|------|------|
| **Camcorder** | 코드 리뷰하는 모습 녹화 |
| **Course** | 1. PR 확인 → 2. 코드 분석 → 3. 피드백 작성 |
| **Cadence** | PR 올라올 때마다 (하루 2회) |
| **Checklist** | ✓ 버그 없음? ✓ 테스트 통과? ✓ 문서 업데이트? |


## 🚀 시작하기

### 단계별 적용 가이드

1. **첫 주**: [Checklist](./checklist.md)만 만들기
   - 가장 쉽고 즉시 효과를 볼 수 있음

2. **둘째 주**: [Course](./course.md) 매뉴얼 작성
   - 체크리스트를 기반으로 상세 단계 정리

3. **셋째 주**: [Cadence](./cadence.md) 설정
   - 언제, 얼마나 자주 할지 명확히

4. **넷째 주**: [Camcorder](./camcorder.md) 영상 촬영
   - 여유가 생기면 영상으로 보완


## 💡 Pro Tips

### Tip 1: 완벽하게 시작하지 마세요
- 초안을 빠르게 만들고 계속 개선하세요
- 80% 완성도로 시작해서 사용하며 보완

### Tip 2: 실제 작업하면서 작성하세요
- 미리 작성하면 실전과 달라질 수 있음
- 작업하면서 동시에 기록

### Tip 3: 팀원 피드백을 받으세요
- 혼자 만든 문서는 나만 이해할 수 있음
- 타인이 따라 할 수 있는지 검증 필수
