## 1일차: 환경 설정하기

### 🎯 목표
작업 환경 세팅 완료하기

### ✅ 할 일

```bash
# 1. Git Worktree 설정 (여러 작업 공간 만들기)
git worktree add ../project-main main
git worktree add ../project-dev develop

# 2. 화면 분할 스크립트 만들기
./vibe-setup.sh
```

### 📋 확인 체크리스트

- [ ] 키보드만으로 영역 이동이 가능한가요?

## 2일차: 첫 작업 해보기

### 🎯 목표
첫 번째 작업을 FCSR 80% 목표로 완료하기

### ✅ 단계별 진행

#### Step 1: Reference 찾기
비슷한 예시 3개 찾기
- Google 검색: "[기능명] reference", "[기능명] example"
- 유명 서비스 둘러보기
- 스크린샷 저장하기

#### Step 2: Prompt 작성
4가지 요소 모두 포함하기
- **Context** (배경): 어떤 프로젝트인지
- **Action** (할 일): 무엇을 만들지
- **Constraint** (제약사항): 어떤 스타일, 규칙으로
- **Output** (결과물): 어떤 파일로 만들지

📖 자세한 내용: [Prompt Engineering 가이드](../03-course-details/04-prompt-engineering.md)

#### Step 3: 코드 생성
한 번에 성공 목표!
- AI에게 명령 입력
- 결과 확인
- 테스트 실행

#### Step 4: Checklist 검증
5가지 모두 체크
- [ ] 창 전환 안 했나요?
- [ ] 레퍼런스 참고했나요?
- [ ] 서버 설정 코드로 저장했나요?
- [ ] 모든 화면 크기에서 정상인가요?
- [ ] 명령을 명확하게 했나요?

📖 자세한 내용: [Checklist 가이드](../02-4c-framework/checklist.md)

### 📋 확인 체크리스트
- [ ] 한 번에 성공했나요?
- [ ] 5가지 체크리스트 모두 통과했나요?

## 3일차: 기록하고 개선하기

### 🎯 목표
내 FCSR 점수 측정하고 개선점 찾기

### ✅ 할 일

```
1. 오늘 완료한 작업 개수 세기
2. 첫 시도에 성공한 작업 개수 세기
3. FCSR 계산하기 (성공 ÷ 전체 × 100)
4. 왜 실패했는지 분석하기
```

### 📝 기록 양식

```markdown
## 나의 FCSR 기록

날짜: 2026-01-17
- 전체 작업: 10개
- 첫 시도 성공: 6개
- FCSR: 60%

### 개선할 점:
- Reference를 충분히 안 찾아봤음
- Prompt가 너무 짧았음
- 화면 전환을 자주 했음

### 다음 목표:
- Reference 3개 이상 찾기
- Prompt 4요소 모두 포함하기
- 화면 분할 상태 유지하기
```

## 💡 자주 묻는 질문

### Q1. 첫날부터 FCSR 80%를 달성해야 하나요?
아니요! 처음엔 40-60%도 괜찮습니다. 중요한 것은 꾸준히 개선하는 것입니다.

### Q2. Git Worktree가 어려워요. 꼭 써야 하나요?
처음엔 화면 분할만 익혀도 됩니다. Git Worktree는 익숙해진 후에 도입하세요.

### Q3. 영상을 꼭 촬영해야 하나요?
개인 학습에는 필수가 아닙니다. 하지만 팀에 공유하려면 영상이 효과적입니다.

### Q4. 하루에 몇 개 작업을 해야 하나요?
개수보다는 FCSR이 중요합니다. 5개를 하더라도 4개 이상은 한 번에 성공하는 게 목표입니다.
