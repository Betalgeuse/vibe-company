### 8.4 Deployment & DevOps

**MVP Deployment Strategy:**

1. **Staging Environment** (AWS EC2 t3.medium)
   - Live replica of production
   - All integration tests run here first
   - Weekly QA sign-off before production deploy

2. **Production Environment** (AWS Auto-Scaling Group)
   - Load balancer (ALB): frontend ‚Üí Node.js instances
   - Min 2 instances (HA), max 5 (scale on CPU > 70%)
   - RDS: Primary + read replica (backup)
   - Redis: Cluster mode, 3 nodes (HA)
   - CDN: CloudFront for static assets + video delivery

3. **CI/CD Pipeline** (GitHub Actions)
   ```yaml
   Push to main ‚Üí 
   ‚îú‚îÄ Unit tests + Lint (5 min)
   ‚îú‚îÄ Deploy to staging (5 min)
   ‚îú‚îÄ Smoke tests (3 min)
   ‚îú‚îÄ Manual approval (‚è≥)
   ‚îî‚îÄ Deploy to production (5 min)
   Total: ~18 min from commit to live
   ```

4. **Database Migrations**
   - Liquibase or Alembic for schema versioning
   - Zero-downtime migrations (backwards-compatible changes first)
   - Weekly backup snapshots (AWS automated)
