### 2️⃣ 진출 전략 (GTM Feasibility): 14/25점 🔴 CRITICAL

#### 약점 2.1: "CAC $2.3K"는 거짓 (Critical)

**제시된 주장**:
- "Founder-led sales로 월 2.5명 고객, CAC $2.3K"

**냉철한 검증**:
```
이론:
  • 주 100명 콜드 이메일
  • 3% 회신율 = 주 3명
  • 50% Demo 전환 = 주 1.5명
  • 30% Close = 월 1.5명... → 계산 맞음?

실제 B2B SaaS 벤치마크:
  • 콜드 이메일 회신율: 1-3% (최고)
  • Demo → Close: 15-30% (SaaS)
  • 실제 Close: 1% (100명 → 1명)
  • 실제 성과: 월 1-2명 (아마도)

Founder 시간 투입:
  • 영업 계획 + 리스트: 주 15시간
  • 콜드 이메일 + 콜드콜: 주 60시간
  • Demo + 협상: 주 20시간
  • 합계: 주 95시간 (거의 풀타임)

월 2.5명 × $2.3K = $5,750 CAC
실제 계산:
  • Founder 월 보수: $5,800 (최소)
  • 고객당 배분: $5,800 / 2.5 = $2,320 ← 이건 "시간 비용만"
  • MVP 개발 중이므로 추가 개발 시간까지 포함하면?
  • 실제 CAC: $4-8K 최소

결론:
  제시된 $2.3K = Founder 시간을 "무료"로 계산한 거짓
  실제 CAC = 최소 $3-5K, 평균 $4-8K
```

**수정된 Unit Economics**:
```
제시된 것:
  CAC $2.3K + LTV $16K = 6.9:1 (좋음)

현실:
  CAC $4-8K + LTV $10K = 1.25-2.5:1 (나쁨)
  CAC Payback: 1개월 → 6-13개월 (훨씬 오래)
```

**영향**: 경제성 붕괴

### 4️⃣ 경쟁력 (Competitive Moat): 14/25점 ❌ WEAK

#### 약점 4.1: 토큰이 차별화가 아님 (Critical)

**우리 주장**:
- "토큰 기반 블록체인 = 차별화"

**고객 현실**:
```
고객(매장주)이 원하는 것:
  "효과적인 재방문 증가" ← Square도 제공

고객이 절대 원하지 않는 것:
  "블록체인", "토큰", "암호화폐", "거래소"

왜?
  • 이해 안 함 (기술 복잡)
  • 위험해 보임 (해킹?)
  • 규제 우려 (라이센스?)
  • 오버스펙 (필요 없음)

포지셔닝:
  Square: "카페 재방문 150% 증가"
  우리: "블록체인 기반 토큰 로열티"
  
  고객 입장:
    Square = 이해하고 믿을 수 있음
    우리 = 복잡하고 불안함

결론:
  토큰 = 우리 입장에서만 "차별화"
  고객 입장에서는 "복잡성만 추가"
```

**영향**: 고객 획득 어려움

## 🔴 Round 2: Opus의 가능한 대응책 검증

### 대응 1: "TAM 재계산했습니다" 

**Opus 주장**: "미국 + 한국 + 동남아 포함하면 $1-2B"

**심사 결과**: ❌ **미충분**
- TAM이 재계산되어도 "경쟁사에 이미 패함" 문제는 해결 안 됨
- 시장이 크면 Square/Toast도 더 강화할 것

---

### 대응 2: "고객 5명 인터뷰 완료, ROI 확인"

**Opus 주장**: "독립 카페 5명 모두 $500/월 ROI 달성"

**심사 결과**: ⚠️ **부분 개선 (하지만 불충분)**
- 5명은 너무 적음 (최소 20-30명 필요)
- "동의" ≠ "실제 구매" (실제 결제 확인 필요)
- Selection bias (친구일 가능성)
- 필요: 30일 이상 실제 사용 중인 고객 2-3개 증명

---

### 대응 3: "CAC $2.3K 맞습니다"

**Opus 주장**: "처음 3명 CAC $1K, 정상화되면 $3-4K"

**심사 결과**: ❌ **거짓 / 미충분**
- CAC는 계속 상승할 것 (초기 고객은 "친구 할인" 패턴)
- 정상화 CAC $4K+는 맞으나, Unit econ이 이미 악화
- LTV:CAC = 7:1이 아니라 2-3:1로 다운그레이드

---

### 대응 4: "토큰이 고객 가치입니다"

**Opus 주장**: "토큰 거래로 고객이 원하는 기능 제공"

**심사 결과**: ❌ **증거 부족**
- 고객이 진짜 "토큰 거래"를 원하는가?
- 인터뷰에서 이게 "구매 이유"였는가?
- 아니면 "좋은 기능"이라고만 했을 뿐인가?
