### **영역 3️⃣: Unit Economics (18/25점) ⚠️ CONDITIONAL**

#### 약점 3.1: LTV Churn 가정이 비현실적 (High)

**제시된 가정**:
```
Churn: 월 5% (주장: 현실적)
Lifespan: 20개월
LTV: $775 × 20 = $15,500
```

**재검증**:
```
실제 SMB SaaS Churn (벤치마크):
  - 전체 SaaS 평균: 월 5-7%
  - SMB 특화: 월 7-10%
  - 카페 전문: 월 10-15% (높음)

왜 높은가?
  1. 카페 폐업율: 월 2-3% (기본)
  2. 로열티 도구 좌절: 사용 안 함 (월 3-5%)
  3. 경쟁사로 이탈: 월 2-3%
  4. 가격 민감도: 월 1-2%
  → 합계: 월 8-13%

Belly 사례:
  - "최고 해결책이었는데 churn 높았음" (= 사용 안 함)
  - 초기 고객 중 70%+ 1년 안에 이탈

우리의 리스크:
  - "또 다른 로열티" = Belly와 동일 위험
  - 고객이 사용하지 않으면 churn 100%

현실적 Churn 재계산:
  - Best case: 월 6%
  - Base case: 월 8-10%
  - Worst case: 월 15%+

LTV 재계산:
  - 월 6% → 17개월 → LTV $13,075
  - 월 8% → 12.5개월 → LTV $9,688
  - 월 10% → 10개월 → LTV $7,750

제시된 LTV ($15,500) vs 현실 ($9,688) = 60% 낙관적
```

**문제**: Churn 과소평가로 LTV 과대평가  
**심각도**: **HIGH** - 경제 모델의 기반이 흔들림  
**영향**: LTV:CAC = 7:1이 아니라 3-4:1로 다운그레이드

#### 약점 3.3: Gross Margin 계산 부실 (Medium)

**제시**: "72% 마진"  
**분석**:
```
제시된 비용:
  - 서버/인프라: 월 $200/고객
  - 지원: 월 $150/고객
  - 합계: 60% COGS (또는 40% margin)

실제 비용:
  - Polygon gas fees: 무시할 수준 (<$0.01/tx)
  - Database: $2K/월 (Supabase) = $40/고객 (50명)
  - Compute: $1K/월 = $20/고객
  - Support: 첫 고객 당 10시간 = $200/고객 (초기)
  - Payment processing: 2.2% (Stripe) = $17/고객 (월 $775 avg)
  - 합계: ~$277/고객

Gross margin 재계산:
  - Revenue: $775/월
  - COGS: $277/월
  - Margin: ($775-$277)/$775 = 64% (아니라 52%)

또한:
  - Support 비용은 고객이 문제 겪을 때 급증 (초기 50%+)
  - 고객이 성공하지 못하면 Support 시간 폭증
  - 실제 Gross Margin: Month 1-2는 음수

현실적 Gross Margin:
  - Month 1-3: 30-40% (높은 onboarding 비용)
  - Month 4+: 50-60% (안정화)
```

**문제**: Support 비용 과소평가, 초기 negative margin  
**심각도**: **MEDIUM** - 캐시플로우 악화  
**영향**: 초기 수익성 더 떨어짐

```
결론:
  토큰 거래소는 우리 입장에서 "차별화"
```

```
  3. 고객 재교육: 0 (사용자는 QR만 스캔)

Switch 하는 데 드는 비용:
  - 설정 시간: 1주 = $0 (점장이 하거나 기술팀)
  - 손실된 거래: 거의 없음 (토큰은 계속 쓸 수 있음)
  - 새 비용: Month 1부터

결론:
  Switch cost << 월 $775
  → 고객은 쉽게 이탈 가능

비교:
  Square = POS 시스템과 통합 → 뺄 수 없음 (High switching)
  우리 = 독립 시스템 → 쉽게 뺄 수 있음 (Low switching)
```

**문제**: Switching cost 매우 낮음  
**심각도**: **HIGH** - Moat 약함  
**영향**: 경쟁사 진입 시 고객 이탈 심화

#### 약점 4.3: Network Effects 거의 없음 (High)

**주장**: "사용자 증가 → 토큰 거래소 가치 증가"  
**분석**:
```
네트워크 효과가 작동하려면:
  1. 참여자가 높을수록 가치 증가
  2. 참여자가 실제로 "상호 작용"

우리의 경우:
  1. 매장주 100명 고객 확보 가정
  2. 고객(소비자) 수: 100 × 1,000 = 100K명 (추정)
  3. 토큰 거래: 누가 누구와?
     - 고객A의 "카페X 토큰" ↔ "카페Y 토큰"?
     - 가능성: 거의 없음 (교환 인센티브 없음)
     - 이미 Square 포인트 → "현금처럼" 쓰고 싶음

네트워크 효과가 없는 이유:
  - 토큰은 "카페 내부" 재방문만 의미
  - 외부 거래 = 사용자가 원하지 않는 기능
  - "블록체인"의 가치를 고객이 모름

실제 사용:
  - 고객A (카페 X): 토큰 쌓아서 할인받음 (거래 X)
  - 고객B (카페 X): 토큰 쌓아서 할인받음 (거래 X)
  - 토큰 거래소: 미사용 (기능 X)
```

**문제**: Network effects가 이론상만 존재  
**심각도**: **HIGH** - Moat 약함  
**영향**: 스케일 시 차별화 사라짐
