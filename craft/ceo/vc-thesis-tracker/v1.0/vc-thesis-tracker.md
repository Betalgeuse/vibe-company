## 📝 Big Ideas 정리 템플릿

새로운 Big Ideas 발표 시 아래 형식으로 정리:

```markdown
# {VC명} Big Ideas {연도}: {파트}

> Source: {URL}
> Date: {발표일}
> Category: {카테고리}

---

## Summary

| # | 아이디어 | 핵심 인사이트 | 내 사업 연관성 |
|---|----------|--------------|---------------|
| 1 | {제목} | {한 줄 요약} | ⭐⭐⭐⭐⭐ |
| 2 | ... | ... | ... |

---

## 상세 분석

### 1. {아이디어 제목} ({저자})
- **핵심**: {핵심 주장}
- **인사이트**: {왜 중요한가}
- **기회**: {창업 기회}
- **연관성**: {내 아이디어와 연결점}

---

## 액션 아이템

- [ ] {후속 조사 필요 항목}
- [ ] {검증 필요 항목}
```

## 🎯 아이디어 검증 시 활용법

### Step 1: 문제 발굴
```
YC RFS → "YC가 원하는 영역인가?"
a16z Big Ideas → "Top VC가 주목하는 트렌드인가?"
```

### Step 2: 시장 검증
```
CB Insights → "관련 스타트업 펀딩 동향"
Crunchbase → "경쟁사 펀딩 규모/시기"
```

### Step 3: 기술 검증
```
AI: OpenAI/Anthropic → "기술적으로 가능한가?"
Crypto: EIPs/Paradigm → "표준/프로토콜 존재하는가?"
```

### Step 4: Validation Quote 추출
```
Big Ideas에서 직접 인용 → 피칭 시 활용
"Sean Neville (Circle 공동창업자): 'KYA가 에이전트 경제의 핵심 인프라'"
```
