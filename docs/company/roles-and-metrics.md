# luerre AI - Roles, R&R, and One Number Metrics

*Based on Dan Martell's "Buy Back Your Time" Replacement Ladder*

---

## Vision & Core Values

**Vision**: Make people do what they love

**Core Values**:
- **No ego, Just impact**: 학벌, 경력, 나이 무관. 실력으로 증명하면 OK
- **Don't (just) get shit done**: 의미 없는 작업은 하지 않는다. 임팩트 있는 일만

**팀 철학**:
- 평생 공부만 했는데 스타트업 씬에 도전해보고 싶은 사람
- The brave dropout - 용기 있게 선택했고, 살아남기 위해 꾸준히 공부하며 희망을 놓지 않는 사람
- 은퇴했지만 7살의 마인드셋으로 새로운 것을 배우고 싶은 사람

## Replacement Ladder (팀 확장 순서)

```
Phase 0: CEO가 모든 것
    ↓
Phase 1: Admin 위임 (행정)
    ↓
Phase 2: Delivery 위임 (실행)
    ↓
Phase 3: Marketing 위임 (마케팅)
    ↓
Phase 4: Sales 위임 (영업)
    ↓
Phase 5: Leadership만 집중
```

## 조직도 확장 로드맵

### Phase 0 (현재)
```
CEO (모든 것)
└── AX Engineer (이상혁)
```

### Phase 1 (+3개월)
```
CEO
├── AX Admin (파트타임)
├── AX Forward Deployed Engineer (대표 겸임)
└── AX Engineer (이상혁)
    └── Junior AX Engineer (인턴 1-2명)
```

### Phase 2 (+6개월)
```
CEO
├── AX Admin
├── AX Forward Deployed Engineer (정식)
└── Lead AX Engineer (이상혁)
    └── Junior AX Engineers (인턴 3-4명)
```

### Phase 3 (+12개월)
```
CEO
├── AX Admin
├── AX Marketing
├── AX Forward Deployed Engineer
│   └── Junior FDE (선택)
└── Lead AX Engineer
    └── AX Engineers (5명+)
```

### Phase 4 (+18개월)
```
CEO (Leadership만)
├── Operations Manager (Admin 확장)
├── AX Marketing
├── AX Sales
├── AX Forward Deployed Engineers (2명+)
└── Engineering Team
    ├── Lead AX Engineer
    └── AX Engineers (다양한 배경)
```

## 직무별 R&R 및 One Number

---

### AX Engineer

**One Number**: 배포된 이슈 수 / 주

| 항목 | 내용 |
|------|------|
| **목표** | 5개/주 (Junior), 8개/주 (Senior) |
| **왜?** | 개발자의 본질 = 동작하는 코드 배포 |
| **측정** | Linear에서 Done 상태로 이동한 이슈 |

**R&R (책임과 역할)**:

| 책임 | 상세 |
|------|------|
| 개발 | 할당된 Linear 이슈 개발 |
| 테스트 | 단위 테스트 작성 및 실행 |
| 배포 | 코드 배포 실행 |
| 코드 리뷰 | 동료 코드 리뷰 참여 |
| 문서화 | 기술 문서 작성 |
| 학습 | 기술 스터디 참여 |

**권한**:
- 기술 스택 제안
- 일정 추정
- 코드 품질 결정

**보조 지표**:
- 버그 재오픈율 < 10%
- 코드 리뷰 통과율 > 90%

---

### Lead AX Engineer

**One Number**: 팀 배포 이슈 수 / 주

| 항목 | 내용 |
|------|------|
| **목표** | 20개/주 (팀 전체) |
| **왜?** | Lead의 본질 = 팀 전체 생산성 |
| **측정** | 팀 전체 Linear Done 이슈 |

**R&R (책임과 역할)**:

| 책임 | 상세 | 비중 |
|------|------|------|
| **Tech Discovery** | X/스레드 모니터링, 새 기술 도입 | 20% |
| 기술 리드 | 아키텍처 결정, 기술 방향 설정 | 25% |
| 코드 리뷰 | 모든 PR 리뷰 및 승인 | 20% |
| 멘토링 | 주니어 개발자 성장 지원 | 15% |
| 품질 관리 | 테스트/배포 프로세스 관리 | 10% |
| 개발 | 핵심/복잡한 기능 직접 개발 | 10% |

**Tech Discovery Cadence**:
- 매일: X/스레드 30분 스캔
- 매주: #tech-radar에 1개 이상 공유
- 격주: Tech Sharing 세션 리드
- 매월: 도구 스택 리뷰

**권한**:
- 기술 스택 최종 결정
- 주니어 업무 배분
- 배포 승인
- 새 도구/기술 도입 결정

**보조 지표**:
- 주니어 성장률 (개인 이슈 수 증가)
- 코드 리뷰 평균 시간 < 4시간
- 팀 버그율 < 10%
- 새 기술 도입 건수 / 분기

### AX Forward Deployed Engineer

**One Number**: 고객 승인된 스펙 수 / 주

| 항목 | 내용 |
|------|------|
| **목표** | 10개/주 |
| **왜?** | FDE의 본질 = 명확한 스펙으로 Scope 확정 |
| **측정** | 고객이 QA 확인 버튼 누른 이슈 수 |

**R&R (책임과 역할)**:

| 책임 | 상세 |
|------|------|
| 요구사항 분석 | 고객 미팅 리드, 요구사항 추출 |
| 프로젝트 설계 | 기술 스택 결정, 아키텍처 설계 |
| 스코프 관리 | Linear 이슈 생성, 우선순위 결정 |
| QA 스펙 확정 | 고객과 QA 기준 합의 |
| 리스크 관리 | 일정/비용 리스크 사전 식별 |
| 고객 커뮤니케이션 | 1차 대응, 진행상황 공유 |
| 견적 | 공수 산정 및 견적서 작성 |

**권한**:
- 프로젝트 스코프 정의
- 우선순위 결정
- 고객 미팅 리드
- 기술 스택 제안

**보조 지표**:
- 스펙 변경 요청률 < 15%
- 고객 미팅 → 이슈화 전환 시간 < 24시간
- 고객 만족도 > 4.5/5

### AX Admin

**One Number**: 대표 시간 절약 / 주

| 항목 | 내용 |
|------|------|
| **목표** | 10시간/주 |
| **왜?** | Admin의 본질 = 대표가 고부가가치 업무에 집중하게 |
| **측정** | 위임된 업무 시간 추적 |

**R&R (책임과 역할)**:

| 책임 | 상세 |
|------|------|
| 일정 관리 | 고객 미팅 스케줄링, 캘린더 관리 |
| 문서 관리 | 계약서, 견적서, 회의록 정리 |
| 회계 기초 | 인보이스 발행, 경비 정리 |
| 채용 어드민 | 지원자 스크리닝, 면접 일정 |
| 도구 관리 | Slack, Notion, Linear 셋업 |
| 커뮤니케이션 | 기본 이메일 대응 |

**권한**:
- 캘린더 관리
- 기본 이메일 대응
- 문서 접근 및 정리

**보조 지표**:
- 행정 오류율 < 5%
- 응답 시간 < 4시간
- 문서 정확도 > 95%

### AX Marketing

**One Number**: Qualified Leads / 월

| 항목 | 내용 |
|------|------|
| **목표** | 20개/월 |
| **왜?** | Marketing의 본질 = 영업 가능한 리드 생성 |
| **측정** | ICP 조건 충족하는 인바운드 리드 |

**ICP (Ideal Customer Profile)**:
- MBA 출신 또는 비기술 창업자
- 예산 있음 (최소 ₩5,000,000)
- 3개월 내 프로젝트 시작 의향
- 웹/앱 개발 필요

**R&R (책임과 역할)**:

| 책임 | 상세 |
|------|------|
| 콘텐츠 제작 | 블로그, 케이스 스터디, SNS |
| MBA 타겟 마케팅 | LinkedIn, 스타트업 커뮤니티 |
| SEO/SEM | 검색 최적화, 광고 운영 |
| 이벤트 | 웨비나, 밋업 기획 |
| 리드 육성 | 뉴스레터, 이메일 시퀀스 |
| 브랜딩 | 회사 이미지 관리 |

**권한**:
- 마케팅 예산 집행
- 콘텐츠 발행
- 채널 선택

**보조 지표**:
- 콘텐츠 발행 수 (주 2개 이상)
- 웹사이트 트래픽 (월 1,000+)
- 리드당 비용 < ₩100,000

### AX Sales

**One Number**: 계약 체결 수 / 월

| 항목 | 내용 |
|------|------|
| **목표** | 3건/월 |
| **왜?** | Sales의 본질 = 돈 들어오게 하기 |
| **측정** | 서명된 계약서 수 |

**R&R (책임과 역할)**:

| 책임 | 상세 |
|------|------|
| 리드 자격검증 | ICP 확인, 예산/타임라인 파악 |
| 첫 미팅 | 니즈 파악, 서비스 소개 |
| 견적 제안 | FDE와 협업하여 제안서 작성 |
| 계약 클로징 | 협상, 계약 체결 |
| 고객 핸드오프 | FDE에게 프로젝트 인계 |
| 업셀/크로스셀 | 기존 고객 추가 프로젝트 |

**권한**:
- 가격 협상 (범위 내)
- 계약 조건 조율
- 고객 미팅 주도

**보조 지표**:
- 리드 → 미팅 전환율 > 30%
- 미팅 → 계약 전환율 > 25%
- 평균 계약 금액 > ₩10,000,000

### Operations Manager (Admin 확장)

**One Number**: 프로세스 자동화율

| 항목 | 내용 |
|------|------|
| **목표** | 80% |
| **왜?** | Ops의 본질 = 시스템이 알아서 돌아가게 |
| **측정** | (자동화된 업무 / 전체 반복 업무) × 100 |

**R&R (책임과 역할)**:

| 책임 | 상세 |
|------|------|
| 프로세스 설계 | 업무 프로세스 표준화 |
| 자동화 | 반복 업무 자동화 구축 |
| 도구 관리 | 전사 도구 통합 관리 |
| 비용 관리 | 운영 비용 최적화 |
| HR 기초 | 급여, 복리후생 관리 |
| 법무 기초 | 계약서 관리, 컴플라이언스 |

**권한**:
- 도구 선택 및 도입
- 프로세스 변경
- 운영 예산 관리

**보조 지표**:
- 운영 비용 효율
- 직원 만족도 > 4/5
- 프로세스 오류율 < 5%

## 측정 방법

### 자동 측정 (Linear 연동)
- **Engineer**: Done으로 이동한 이슈 자동 카운트
- **FDE**: "QA Approved" 라벨 붙은 이슈 카운트

### 수동 측정 (초기)
- **Admin**: 대표가 위임한 업무 시간 기록
- **Sales**: CRM 또는 스프레드시트
- **Marketing**: Google Analytics + 리드 폼

### 미래 (AX FDE 플랫폼)
- 모든 지표 자동 수집 및 대시보드화
- 주간 리포트 자동 생성
- 알림 및 경고 시스템
