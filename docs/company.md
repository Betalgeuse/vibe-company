## Problem: 외주 개발은 왜 실패하는가?

### 1. 끝없는 요구사항 변경 (Scope Creep)
비기술 창업자와 개발팀 간의 커뮤니케이션 부재로 인해 프로젝트 범위가 계속 확장됩니다.

**주요 현상**:
- "이것도 추가해주세요"
- "그건 당연히 포함된 거 아닌가요?"
- 명확한 QA 스펙 부재로 무한 수정 요청
- 프로젝트 일정 지연과 예산 초과

**고객사 사례**:
MBA 출신 창업자 A씨는 외주 개발사와 "간단한 커머스 사이트"를 계약했으나, 개발 중 "이건 당연히 되어야 하는 기능 아닌가요?"라는 질문을 반복하며 프로젝트 범위가 2배로 늘어났습니다. 결국 예산 초과와 일정 지연으로 프로젝트가 중단되었습니다.

### 2. 불투명한 진행 상황
고객은 개발이 어디까지 진행되었는지, 언제 끝날지 알 수 없습니다.

**주요 현상**:
- 회의록은 있지만 실제 액션 아이템은 불명확
- 구두 약속과 실제 deliverable의 불일치
- 진행 상황 확인을 위한 또 다른 회의 필요
- "거의 다 됐어요"라는 모호한 답변

**고객사 사례**:
스타트업 B사는 매주 개발 미팅을 진행했지만, 실제로 무엇이 완료되었고 무엇이 남았는지 파악할 수 없었습니다. 회의록에는 "검토 중", "진행 중"이라는 모호한 표현만 가득했고, 결국 예상 런칭일을 3개월 넘기게 되었습니다.

### 3. 커뮤니케이션 오버헤드
개발자는 개발보다 회의와 설명에 더 많은 시간을 씁니다.

**주요 현상**:
- "이게 왜 안 되나요?"라는 반복적인 질문
- 기술적 배경이 없는 고객에게 설명하는 시간
- 문서화되지 않은 암묵적 요구사항
- 개발 시간보다 커뮤니케이션 시간이 더 많음

**개발자 사례**:
외주 프로젝트를 진행하던 개발자 C씨는 실제 코딩 시간보다 고객과의 회의, 수정 요청 대응, 설명 등에 더 많은 시간을 소비했습니다. "제대로 된 개발을 하나도 못하고 있다"는 불만과 함께 번아웃을 경험했습니다.

## Solution: AX FDE - AI가 관리하는 투명한 외주 개발

### luerre AI의 차별점

luerre AI는 단순한 외주 개발 회사가 아닙니다. **AX FDE**라는 AI 시스템이 프로젝트의 모든 커뮤니케이션과 요구사항을 자동으로 관리하여, 고객과 개발자 모두가 본질적인 일에만 집중할 수 있게 합니다.

### 핵심 솔루션

#### 1. 자동 요구사항 구조화
**문제**: 회의에서 나온 애매한 요구사항들이 실제 작업으로 연결되지 않음

**해결**:
- 모든 고객 미팅을 자동 녹음 및 전사
- AX FDE가 대화에서 요구사항을 자동 추출
- Linear 스타일로 이슈화 (제목, 설명, 우선순위, 예상 공수)
- 고객이 "QA 확인" 버튼으로 스펙 확정
- 확정된 스펙만 개발 진행

**결과**:
- "이건 당연히 포함된 거 아닌가요?" → 모든 요구사항이 이슈로 기록되어 투명하게 관리
- 무한 수정 요청 차단 → 새로운 요구사항 = 새로운 이슈 = 명확한 공수 산정

#### 2. 투명한 대시보드
**문제**: 고객은 진행 상황을 알 수 없고, 또 회의를 잡아야 함

**해결**:
- 실시간 프로젝트 진행 상황 대시보드
- 각 이슈별 상태 (To Do / In Progress / In Review / Done)
- 타임라인 및 예상 완료일
- 모든 회의록과 결정 사항 히스토리 보관
- 궁금한 점은 대시보드에서 직접 질문 → AI가 1차 답변

**결과**:
- 24/7 진행 상황 확인 가능
- 회의 횟수 80% 감소
- "거의 다 됐어요" → 정확한 완료율과 예상 완료일 제공

#### 3. Scope Creep 방지
**문제**: "이것도 당연히 포함된 거 아닌가요?"라는 무한 추가 요청

**해결**:
- 프로젝트 시작 시 명확한 스코프 정의
- 새로운 요구사항 = 새로운 이슈 생성 + 공수 산정
- 확정되지 않은 요구사항은 개발하지 않음
- 변경사항은 타임라인과 비용에 자동 반영
- 모든 것이 투명하게 기록됨

**결과**:
- 프로젝트 일정 준수율 95% 이상
- 예산 초과 0건
- 고객과의 분쟁 방지

#### 4. 개발자는 오직 개발에만
**문제**: 개발자가 회의, 설명, 문서화에 시간을 뺏김

**해결**:
- AX FDE가 고객 커뮤니케이션 1차 처리
- 구조화된 이슈만 개발자에게 전달
- 기술적 질문도 AI가 1차 필터링
- 개발자는 Linear 스타일 이슈만 보고 코딩
- 코드 리뷰 및 배포도 자동화

**결과**:
- 개발자 생산성 3배 증가
- 개발자 만족도 향상
- 저인건비 학생 인턴도 프로처럼 일할 수 있는 환경

### luerre AI 프로세스

```
고객 미팅 
  ↓ (자동 녹음 및 전사)
AX FDE가 요구사항 추출
  ↓ (Linear 스타일 이슈화)
고객이 QA 확인 버튼으로 확정
  ↓ (스펙 확정)
AX Engineer가 개발 진행
  ↓ (실시간 대시보드 업데이트)
고객이 진행 상황 확인 (24/7)
  ↓ (배포)
명확한 deliverable 전달
```

### 기술 스택 (Vibers Inc 계승 및 발전)
- **AI/ML**: 음성 인식, NLP 기반 요구사항 추출, 자동 이슈 생성
- **플랫폼**: 웹 기반 대시보드, 실시간 동기화, Linear 스타일 UI/UX
- **개발 자동화**: CI/CD, 자동 테스트, 배포 파이프라인
- **협업 도구 통합**: Slack, Notion 등과 연동
