## 9. Risk Assessment & Mitigation

| Risk | Likelihood | Impact | Mitigation |
|------|------------|--------|------------|
| **Long Sales Cycle** | Medium (45-60 days typical for compliance) | High (cash flow squeeze) | Offer annual contract at discount (15% off). Freemium creates 12-month engagement before close. |
| **Competing with Incumbents (Visual Compliance)** | Medium | Medium | Freemium + SMB-first pricing undercuts incumbents. Faster onboarding. Position as "distributed" (API-first) vs "enterprise-only." |
| **CSL List Data Errors** | Low (government maintains) | High (false positives destroy trust) | Implement fuzzy matching to reduce false positives. Show confidence scores. Always allow manual override. Monitor for CSL updates. |
| **Regulatory Changes (New sanctions, BIS rules)** | Medium | Medium | Monitor BIS updates daily. Communicate changes to customers via email/in-app. Build "future-proof" architecture (easy to add new lists). |
| **Product-Market Fit Misses** | Medium | High (wasted 3 months) | Validate ICP with 20 discovery calls before launch. Test messaging variants. Pivot quickly if response is poor. |
| **Technical Debt (Poor Architecture)** | Medium (MVP-quality code) | Medium | Design for multi-tenancy from Day 1 (even if not needed). Use established frameworks (not custom). Plan refactor in Q2. |
| **Churn Due to Integrations** | Medium (customers want API integration) | Medium | Roadmap API in Q2 (P1 priority). Offer CSV as bridge solution. |
| **Team Fatigue (Solo founder or small team)** | Medium | High | Hire SDR by Month 4 to reduce founder sales load. Use no-code tools (Zapier, Make) to avoid over-engineering. |
| **Paid Search CAC Too High** | Low (calculated at $300-400) | Medium | Start with organic (blog, content). Test paid only after organic works. Set CAC cap at $800. |
